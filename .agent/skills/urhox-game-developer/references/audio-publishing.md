## 1  Audio Integration

### Quick AudioManager Setup
```lua
-- scripts/game/AudioManager.lua (use audio-manager skill for full code)
local Audio = require("game.AudioManager")

function Start()
    Audio.Init(scene_)
    -- Optional BGM:
    Audio.PlayBGM("audio/bgm/main_theme.ogg", 1.5)  -- 1.5s crossfade
end

function HandleUpdate(et, ed)
    Audio.Update(ed:GetFloat("TimeStep"))   -- drives BGM crossfade
end
```

### SFX One-Shot Pattern
```lua
-- Creates a temporary node, plays sound, auto-removes node on finish
Audio.Play("audio/sfx/jump.ogg")           -- default volume
Audio.Play("audio/sfx/explosion.ogg", 0.8) -- 80% volume

-- Sound type routing:
-- "Music"  → BGM, affected by BGM volume slider
-- "Effect" → SFX, affected by SFX volume slider
```

### Audio File Guidelines
| Spec | Recommendation |
|------|---------------|
| Format | OGG Vorbis (best compression/quality) |
| BGM bitrate | 128–192 kbps |
| SFX bitrate | 64–96 kbps |
| BGM looping | Set `snd.looped = true` |
| Silence trim | Remove leading silence for SFX |
| Asset path | `assets/audio/sfx/name.ogg` → reference as `"audio/sfx/name.ogg"` |

### Volume Architecture
```lua
Audio.SetBGMVolume(0.7)   -- 0.0–1.0
Audio.SetSFXVolume(1.0)   -- 0.0–1.0
Audio.SetMute(true)       -- mutes all

-- Per-type gain via UrhoX subsystem:
audio:SetMasterGain("Music",  0.7)
audio:SetMasterGain("Effect", 1.0)
```

### 3D Spatial Audio
```lua
-- Read: engine-docs/api/audio.md → SoundSource3D
local node = scene_:CreateChild("Enemy")
local src3d = node:CreateComponent("SoundSource3D")
src3d.soundType   = "Effect"
src3d.nearDistance = 2.0
src3d.farDistance  = 20.0
src3d:Play(cache:GetResource("Sound", "audio/sfx/engine.ogg"))

-- Listener (usually camera or player node)
local listenerNode = scene_:CreateChild("Listener")
listenerNode:CreateComponent("SoundListener")
audio.listener = listenerNode:GetComponent("SoundListener")
```

### Notification Sound Pattern
```lua
-- Map notification type to SFX
function Audio.PlayNotify(ntype)
    local map = {
        success = "audio/sfx/notify_success.ogg",
        error   = "audio/sfx/notify_error.ogg",
        warn    = "audio/sfx/notify_warn.ogg",
    }
    Audio.Play(map[ntype] or "audio/sfx/notify_info.ogg")
end
```

## 2  Asset Pipeline

### Directory Structure
```
workspace/
  assets/
    Textures/       -- PNG/JPG textures (reference without "assets/" prefix)
    Models/         -- MDL 3D models
    Sounds/         -- legacy path (prefer audio/)
    audio/
      bgm/          -- background music OGG
      sfx/          -- sound effects OGG
    Fonts/          -- TTF/OTF fonts
    image/          -- NanoVG images (PNG)
    Particle/       -- particle effect XML
    Materials/      -- material XML
    Techniques/     -- shader technique XML
```

### Resource Reference Rules
```lua
-- Assets root = assets/ AND scripts/
-- Reference WITHOUT the root prefix:
cache:GetResource("Texture2D",  "Textures/hero.png")
cache:GetResource("Model",      "Models/hero.mdl")
cache:GetResource("Sound",      "audio/sfx/jump.ogg")
cache:GetResource("Material",   "Materials/hero.xml")

-- NanoVG image:
nvgCreateImage(vg_, "image/icon.png", 0)
```

### Asset Optimization Checklist
- [ ] Textures: power-of-2 dimensions (256, 512, 1024, 2048)
- [ ] Textures: compress to JPG for photos, PNG for sprites with transparency
- [ ] Audio: OGG at appropriate bitrate, silence trimmed
- [ ] Models: face count ≤ 5000 for mobile, use LOD for > 10000
- [ ] Particle effects: max particles ≤ 200 on mobile

### Download While Playing (DWP)
```lua
-- For large assets not needed immediately:
-- Read: engine-docs/recipes/download-while-playing.md

-- Manual download trigger:
cache:BackgroundLoadResource("Texture2D", "Textures/level2_bg.png",
    false, nil)

-- Check when done:
if cache:GetResource("Texture2D", "Textures/level2_bg.png") then
    -- ready
end
```

## 3  Build Configuration

### project.json Key Fields
```json
{
  "project_id": "<auto-generated>",
  "entry": "main.lua",
  "version": "1.0.0",
  "taptap_publish": {
    "title": "My Game",
    "category": "casual",
    "screen_orientation": "landscape",
    "description": "Game description"
  },
  "assets": {
    "icon": "./game_material/icon.png",
    "screenshots": [
      "./game_material/screenshot1.png",
      "./game_material/screenshot2.png",
      "./game_material/screenshot3.png"
    ],
    "promotional_image": "./game_material/promo.png"
  }
}
```

### Multiplayer Build Config
```json
{
  "@runtime": {
    "multiplayer": {
      "enabled": true,
      "max_players": 8,
      "match_info": {
        "desc_name": "free_match",
        "player_number": 2
      }
    }
  }
}
```

### Build Tool Usage
```
Always call the UrhoX MCP build tool after code changes.
Never preview without building first.

Entry variants:
- Single-player: entry = "main.lua"
- Multiplayer: entry_client = "client_main.lua", entry_server = "server_main.lua"
```

## 4  i18n & Localization

### Text Translation
```lua
-- Read: engine-docs/recipes/i18n-translation.md
-- 1. Enable in .project/i18n.json:
{
  "enabled": true,
  "source_lang": "zh",
  "target_langs": ["en", "ja"]

-- 2. Run i18n_extract MCP tool to generate pending translation files
-- 3. Fill in translations
-- 4. Build — engine substitutes keys at runtime
```

```lua
-- Read: engine-docs/recipes/i18n-resource.md
-- File naming convention:
-- assets/Textures/title.png       -- default (zh)
-- assets/Textures/title@en.png    -- English variant
-- assets/Textures/title@ja.png    -- Japanese variant

-- Engine routes automatically at runtime based on device language
```

## 5  TapTap Publishing

### Required Assets
| Asset | Size | Notes |
|-------|------|-------|
| Icon | 512×512 PNG | No text recommended |
| Screenshots | min 3, 1920×1080 or 1080×1920 | Real gameplay only |
| Promo image | 1920×1080 PNG | Marketing banner |

### Publishing Checklist
- [ ] `build` tool run successfully
- [ ] `project.json` has `taptap_publish.title` and `category`
- [ ] `screen_orientation` matches actual game
- [ ] Icon generated via `generate_game_material` MCP (ICON type)
- [ ] 3+ screenshots from real gameplay (not AI-generated mockups)
- [ ] Promotional image generated
- [ ] `publish_to_taptap` MCP called with explicit user confirmation

### Category Values
`rpg` `casual` `action` `strategy` `simulation` `trivia` `arcade`
`adventure` `card` `sports` `racing` `puzzle` `educational` `music`
`word` `board`

### Test Before Publishing
```
1. Run build tool
2. Call generate_test_qrcode MCP
3. Scan QR on mobile device
4. Verify gameplay, audio, UI layout
5. Use add_test_whitelist MCP to invite testers
```

### Generating Game Materials
```
generate_game_material MCP:
- material_type: "ICON"                 -- no input images needed
- material_type: "PROMO"                -- needs real screenshots
- material_type: ["PROMO", "SCREENSHOT"] -- batch generate
- material_type: "SQUARE_PROMO"         -- 1:1 video cover
- material_type: "COVER_HORIZONTAL"     -- PC horizontal cover (needs PROMO first)
```

## 6  GameJam Submission

### Binding to a GameJam Event
```
1. Ensure project is built (build tool called)
2. Ensure taptap_publish.title and .category are set in project.json
3. Call bind_game_jam MCP with game_jam_event_name = "<event name>"
   - Tool fuzzy-searches for the event
   - Creates TapTap App if not yet created
   - Binds game to the event
```

### GameJam Readiness Checklist
- [ ] Core gameplay loop complete and fun
- [ ] Win/lose state implemented
- [ ] At least one level / scene
- [ ] BGM or ambient audio present
- [ ] Icon and screenshots generated
- [ ] No crash on start
- [ ] Game runs on mobile (test via QR code)
- [ ] Description explains the game in ≤ 100 words

### Common Rejection Reasons
| Issue | Fix |
|-------|-----|
| AI-generated screenshots submitted as gameplay | Use real in-game captures |
| Missing icon | Generate via `generate_game_material` ICON type |
| Fewer than 3 screenshots | Capture more gameplay moments |
| Crash on launch | Fix all LSP errors, run build, test via QR |
| Orientation mismatch | Set `screen_orientation` to match actual layout |
