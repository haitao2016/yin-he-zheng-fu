# 发布前全面质量检查

## 概述

本 Skill 是发布前的**终极质检关卡**，整合并超越 `game-bug-checker`（代码BUG）和 `device-adaptation-bug-fixer`（设备适配）的能力，扩展至 **7 大维度、50+ 条检测规则**，覆盖从代码正确性到用户体验的全链路。

### 与其他 Skill 的关系

| Skill | 覆盖范围 | 本 Skill 覆盖？ |
|-------|---------|:---:|
| `game-bug-checker` | 17 类引擎 API 误用 | 完全包含 (维度4) |
| `device-adaptation-bug-fixer` | 20 类跨设备适配 | 完全包含 (维度1/2/3) |
| **pre-release-check** | 50+ 条 7 维度全面检查 | 本体 |

**本 Skill 在触发时直接执行完整检查，不再分别调用上述两个 Skill。**

---

## 执行流程

```
PREPARE → SCAN(7维) → CLASSIFY → FIX/FLAG → REPORT → BUILD
   │          │           │          │          │        │
   │          │           │          │          │        └─ 调用 build 工具验证
   │          │           │          │          └─ 输出完整诊断报告
   │          │           │          └─ 自动修复 / 人工标记
   │          │           └─ CRITICAL > HIGH > MEDIUM > LOW
   │          └─ 按 7 大维度逐维扫描
   └─ 收集文件列表 + 读取项目配置
```

### 步骤 0: PREPARE — 准备

```bash
# 收集待扫描文件
SCRIPTS=$(find scripts/ -name "*.lua" -type f 2>/dev/null)
FILE_COUNT=$(echo "$SCRIPTS" | wc -l)
LINE_COUNT=$(find scripts/ -name "*.lua" -exec cat {} + 2>/dev/null | wc -l)

# 读取项目配置
cat .project/settings.json 2>/dev/null  # 多人/单机模式
cat .project/project.json 2>/dev/null   # 发布配置
```

### 步骤 1-7: SCAN — 按 7 大维度扫描

逐维度执行检测规则（详见 [references/check-rules.md](references/check-rules.md)），每条规则标记为：
- **🔧 AUTO-FIX**: 可安全自动修复
- **🔍 MANUAL**: 需人工确认
- **📋 INFO**: 信息提示，非必须修复

### 步骤 8: FIX — 自动修复

1. 先展示所有 AUTO-FIX 项的修复预览
2. 用户确认后批量执行
3. MANUAL 项逐个向用户展示修复建议

### 步骤 9: REPORT — 输出报告

格式见下方「诊断报告格式」。

### 步骤 10: BUILD — 重新构建

所有修复完成后，调用 `build` 工具重新构建项目，验证修复无误。

## 自动修复安全策略

1. **仅修改 `scripts/`** 下用户代码
2. 修复前展示**修复前后对比**
3. 在修复行上方添加注释 `-- [PRE-REL:Dx-xx]`
4. 注释/字符串中的代码不修复
5. 语义不确定的变量不修复
