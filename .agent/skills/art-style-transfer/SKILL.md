# 游戏素材艺术风格迁移系统

> 基于 Neural Style Transfer 理念，用 AI 图像工具链将任意艺术风格系统性地应用到全套游戏素材。

---

## 1. 身份与原理

### 1.1 核心理念

本 Skill 将 Neural Style Transfer（神经风格迁移）论文的核心思想——**分离内容与风格，将风格从参考图迁移到目标图**——适配到 UrhoX 游戏素材生产流程中。

原始论文（Gatys et al., 2015）使用 VGG 卷积网络分离内容特征和风格特征，然后在像素空间优化合成。本 Skill 不直接运行神经网络，而是利用引擎内置的 AI 图像工具（`edit_image`、`generate_image`）实现等效效果：

```
内容图（游戏素材） + 风格图（艺术参考） → AI 编辑 → 风格化素材
```

### 1.2 与 Neural Style Transfer 的映射

| 原始论文概念 | 本 Skill 对应实现 |
|------------|-----------------|
| Content Image（内容图） | 用户的原始游戏素材 |
| Style Image（风格图） | 用户指定的艺术风格参考图 |
| Content Weight（内容权重） | edit_image prompt 中对保留原始内容的描述强度 |
| Style Weight（风格权重） | edit_image prompt 中对风格迁移的描述强度 + reference_images |
| Style Scale（风格尺度） | 控制风格迁移的粒度（全局色调 vs 笔触细节） |
| Total Variation（平滑度） | prompt 中对均匀性/平滑度的描述 |
| 迭代优化 | 多轮 edit_image 逐步逼近目标风格 |

### 1.3 与相关 Skill 的关系

| Skill | 职责 | 与本 Skill 的关系 |
|-------|------|-----------------|
| **art-style-transfer**（本 Skill） | 风格统一性迁移 | 核心 |
| `iterative-image-craft` | 单张图像迭代打磨 | 可作为下游精修工具 |
| `ai-asset-pipeline` | 多资产批量管线 | 可作为上游编排调度 |
| `pixel-art-generator` | 像素风专用生成 | 像素风格是本 Skill 支持的风格之一 |
| `@m-mikoto_character-portraits` | 角色立绘批量生成 | 立绘风格化是本 Skill 的应用场景之一 |

## 2. 四步工作流

```
DEFINE（定义风格） → ANCHOR（锚定参考） → TRANSFER（批量迁移） → VERIFY（一致性校验）
```

### 步骤 1: DEFINE — 定义目标风格

#### 2.1.1 风格来源（三选一）

| 来源 | 描述 | 操作 |
|------|------|------|
| **风格参考图** | 用户提供一张画作/截图 | 分析风格要素，生成风格描述 |
| **风格名称** | 用户说"油画风""赛博朋克" | 从内置风格库匹配风格描述 |
| **自定义描述** | 用户详细描述想要的风格 | 直接使用用户描述 |

#### 2.1.2 风格六维度分析

无论来源如何，都需要提取/定义以下六个维度：

```
1. 色彩基调（Palette）      → 暖色/冷色/高饱和/低饱和/单色/互补色
2. 笔触质感（Texture）      → 厚涂/薄涂/平涂/颗粒/光滑/数字化
3. 光影风格（Lighting）     → 硬光/柔光/平光/戏剧光/环境光遮蔽
4. 线条特征（Linework）     → 粗轮廓/细线/无线/手绘/几何
5. 细节层次（Detail Level） → 极简/适中/繁复/超写实
6. 整体氛围（Mood）         → 明亮/阴暗/梦幻/复古/未来/温馨
```

#### 2.1.3 风格描述模板

分析完成后，生成标准化风格描述（用于后续所有迁移操作的 prompt 基础）：

```
风格描述模板:
"{palette}色调，{texture}笔触质感，{lighting}光影，
{linework}线条风格，{detail}细节层次，{mood}整体氛围，
{additional_keywords}"

示例:
"暖色调高饱和，厚涂油画笔触质感，戏剧性侧光光影，
粗轮廓线条，适中细节层次，复古温馨氛围，
印象派风格"
```

### 步骤 2: ANCHOR — 锚定风格参考

#### 2.2.1 风格锚定图

为确保批量迁移的风格一致性，需要建立"风格锚定"：

1. **如果用户提供了风格参考图**：直接作为锚定图
2. **如果用户只指定了风格名称**：用 `generate_image` 生成一张典型的风格样本作为锚定图

```
锚定图生成 prompt:
"[风格名称]风格的典型画作示例，展示该风格的色彩、笔触、
光影和构图特征，纯艺术展示，无特定主题，
{style_description}"

参数:
- target_size: "512x512"（锚定图不需要大尺寸）
- aspect_ratio: "1:1"
- thinking_level: "high"（需要高质量风格表现）
```

#### 2.2.2 风格测试

用锚定图对一个简单素材做测试迁移，让用户确认风格效果：

```
测试迁移流程:
1. 选取用户的一个代表性素材
2. 用 edit_image 执行风格迁移
3. 展示原图 vs 风格化结果
4. 用户确认或调整
```

### 步骤 3: TRANSFER — 执行风格迁移

#### 2.3.1 单张素材迁移

对每张素材调用 `edit_image`：

```
工具: edit_image
参数:
  image: "原始素材路径"
  prompt: "{transfer_prompt}"（见 §3 分品类 prompt 模板）
  reference_images: ["风格锚定图路径"]
  target_size: "与原图相同或指定尺寸"
  aspect_ratio: "与原图匹配"
  name: "styled_{原始文件名}"
  thinking_level: "high"
```

#### 2.3.2 批量素材迁移

当素材数量 > 1 时，使用 `batch_generate_images` 并行处理：

```
工具: batch_generate_images
参数:
  images: [
    {
      prompt: "{transfer_prompt_for_asset_1}",
      reference_images: ["风格锚定图路径", "原始素材1路径"],
      target_size: "512x512",
      name: "styled_asset_1"
    },
    {
      prompt: "{transfer_prompt_for_asset_2}",
      reference_images: ["风格锚定图路径", "原始素材2路径"],
      target_size: "512x512",
      name: "styled_asset_2"
    },
    ...
  ]
```

**注意**: `batch_generate_images` 使用 `generate_image`（不是 `edit_image`），
因此对于需要精确保留内容的素材，仍需逐张使用 `edit_image`。
`batch_generate_images` 更适合从零生成同风格的新素材。

#### 2.3.3 迁移策略选择

| 场景 | 工具 | 内容保留度 | 风格迁移强度 |
|------|------|----------|------------|
| **精确迁移**（保留原素材内容） | `edit_image` | 高 | 中 |
| **重绘迁移**（以风格为主） | `edit_image` | 低 | 高 |
| **新生成**（全新同风格素材） | `generate_image` | 无 | 全 |
| **批量新生成** | `batch_generate_images` | 无 | 全 |

### 步骤 4: VERIFY — 一致性校验

迁移完成后，执行以下检查：

#### 2.4.1 视觉一致性清单

```
□ 色彩基调统一（无突兀的色温差异）
□ 笔触质感统一（不存在部分素材平涂而部分厚涂）
□ 光影方向统一（不存在光源方向矛盾）
□ 细节层次统一（不存在部分极简而部分繁复）
□ 整体氛围统一（不存在情绪矛盾）
```

#### 2.4.2 技术规格清单

```
□ 输出尺寸符合引擎要求（2 的幂次推荐：256/512/1024）
□ 透明通道保留正确（图标/UI 元素需保留透明度）
□ 文件命名规范（styled_ 前缀）
□ 文件存放在 assets/ 目录下
```

#### 2.4.3 不一致素材处理

如果发现某些素材与整体风格不一致：
1. 标记不一致的素材
2. 调整该素材的 prompt（增强或减弱特定风格维度）
3. 重新迁移
4. 再次对比

## 3. 分品类 Prompt 模板

### 3.1 纹理贴图（Textures）

适用于：地面、墙壁、材质贴图

```
精确迁移 prompt:
"将这张纹理贴图转换为{style_description}风格，
保留纹理的基本图案和结构，
改变色彩基调为{palette}，
添加{texture}笔触质感，
保持纹理的可平铺性，
适合作为游戏 3D 模型的表面贴图"

关键参数:
  transparent: false
  target_size: "512x512" 或 "1024x1024"（纹理推荐 2 的幂次）
  aspect_ratio: "1:1"（纹理通常正方形）
```

**重要**：纹理贴图的风格迁移需要注意可平铺性（tileable），prompt 中必须强调。

### 3.2 UI 界面（UI Elements）

适用于：按钮、面板、边框、装饰

```
精确迁移 prompt:
"将这个 UI 元素转换为{style_description}风格，
保留元素的基本形状和布局，
色调调整为{palette}，
添加{texture}质感的边框和装饰，
保持元素的清晰可读性和交互可识别性，
保持透明背景"

关键参数:
  transparent: true（UI 元素通常需要透明背景）
  target_size: 根据 UI 元素尺寸
```

**注意**：UI 元素的风格迁移强度应适中，不能影响可读性和可交互性。

### 3.3 角色立绘（Character Art）

适用于：角色头像、全身立绘、NPC 图像

```
精确迁移 prompt:
"将这个角色转换为{style_description}风格，
保留角色的面部特征、姿势和服装设计，
用{palette}色调重新着色，
以{texture}笔触绘制，
{lighting}光影效果，
{linework}线条风格，
角色特征清晰可辨认，
保持透明背景"

关键参数:
  transparent: true
  aspect_ratio: "2:3" 或 "3:4"（角色立绘通常竖版）
  thinking_level: "high"（角色需要高质量）
```

### 3.4 场景背景（Backgrounds）

适用于：关卡背景、主菜单背景、过场图

```
精确迁移 prompt:
"将这个场景转换为{style_description}风格，
保留场景的空间构图和主要元素，
整体色调调整为{palette}，
用{texture}笔触重新绘制，
{lighting}光影氛围，
{mood}整体感觉，
适合作为游戏场景背景"

关键参数:
  transparent: false
  aspect_ratio: "16:9" 或 "3:2"（背景通常横版）
  target_size: "1024x512" 或 "1344x768"
```

### 3.5 图标道具（Icons & Items）

适用于：物品图标、技能图标、道具

```
精确迁移 prompt:
"将这个图标/道具转换为{style_description}风格，
保留物品的基本外形和辨识度，
用{palette}色调着色，
{texture}质感表面，
简洁清晰，
适合作为游戏物品图标，
保持透明背景"

关键参数:
  transparent: true
  target_size: "128x128" 或 "256x256"
  aspect_ratio: "1:1"
```

## 5. 风格迁移的控制参数

### 5.1 内容保留度（Content Preservation）

通过 prompt 措辞控制保留多少原始内容：

| 级别 | prompt 关键词 | 效果 |
|------|-------------|------|
| **高保留** | "保留原始图像的所有细节和构图，仅改变色调和质感" | 内容几乎不变，仅风格化表面 |
| **中保留** | "保留基本形状和布局，用新风格重新诠释" | 可辨认原始内容，风格明显改变 |
| **低保留** | "以原图为灵感，用新风格完全重新创作" | 仅保留主题概念，大幅风格化 |

### 5.2 风格强度（Style Intensity）

| 级别 | prompt 策略 | reference_images 用法 |
|------|-----------|---------------------|
| **轻度** | 仅描述色调变化 | 不传或 1 张风格参考 |
| **中度** | 描述色调 + 笔触 + 光影 | 1 张风格参考 |
| **强度** | 完整六维度描述 | 1-2 张风格参考 |

### 5.3 迁移粒度（Transfer Granularity）

| 粒度 | 描述 | 适用场景 |
|------|------|---------|
| **全局** | 整体色调和氛围迁移 | 场景背景、大面积纹理 |
| **局部** | 保留结构，迁移表面质感 | 角色、UI 元素 |
| **混合** | 主体保留，背景强迁移 | 需要主体突出的构图 |

## 6. 操作指南

### 6.1 触发后的执行流程

```
收到风格迁移请求
  ↓
判断风格来源（参考图/名称/描述）
  ├─ 参考图 → 分析六维度 → 生成风格描述
  ├─ 名称   → 从 §4 内置风格库匹配
  └─ 描述   → 直接使用用户描述
  ↓
生成/确认风格锚定图
  ↓
向用户展示锚定图，确认风格方向
  ↓
收集待迁移素材清单
  ↓
分品类选择 prompt 模板（§3）
  ↓
选择迁移策略（精确/重绘/新生成）
  ↓
执行迁移
  ├─ 单张 → edit_image
  ├─ 多张精确 → 逐张 edit_image
  └─ 多张新生成 → batch_generate_images
  ↓
展示结果，执行一致性校验
  ↓
标记不一致素材 → 调整后重新迁移
  ↓
确认完成
```

### 6.2 与用户的交互要点

1. **首先确认风格方向**：不要跳过 DEFINE 和 ANCHOR 步骤
2. **先做测试迁移**：选一张有代表性的素材做测试，确认后再批量执行
3. **展示对比**：始终展示"原图 vs 风格化"的对比
4. **提供调整选项**：告知用户可以调整内容保留度和风格强度
5. **标注不确定项**：如果风格迁移效果可能不理想（如纹理平铺性），主动提醒

## 7. 质量保障

### 7.1 常见问题与解决

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| 风格化后内容面目全非 | 风格强度过高 | 提高内容保留度，减弱 prompt 中的风格描述 |
| 风格化效果不明显 | 风格强度过低 | 增加风格描述维度，添加 reference_images |
| 批量素材风格不统一 | 缺少风格锚定 | 确保所有迁移使用同一锚定图作为 reference_images |
| 纹理失去平铺性 | prompt 未强调 | 在 prompt 中加入"保持纹理可平铺，无缝衔接" |
| 透明通道丢失 | 未设置 transparent | 对需要透明背景的素材设置 transparent: true |
| UI 元素可读性下降 | 风格化过度 | 降低 UI 元素的风格迁移强度，优先保持清晰度 |
| 角色辨识度降低 | 面部特征被改变 | prompt 中强调"保留面部特征和表情" |

### 7.2 引擎集成注意事项

```lua
-- 风格化素材在引擎中的引用方式与普通素材完全一致
-- 只需将风格化文件放到 assets/ 对应子目录

-- 纹理
local texture = cache:GetResource("Texture2D", "styled/textures/ground_styled.png")

-- UI 图片
UI.Image { source = "styled/ui/button_bg_styled.png" }

-- 背景
local bgTexture = cache:GetResource("Texture2D", "styled/backgrounds/level1_bg_styled.png")
```

### 7.3 尺寸建议

| 素材类型 | 推荐 target_size | aspect_ratio |
|---------|-----------------|--------------|
| 纹理贴图 | 512x512 / 1024x1024 | 1:1 |
| UI 按钮/面板 | 256x128 / 512x256 | 根据实际 |
| 角色头像 | 256x256 | 1:1 |
| 角色立绘 | 512x768 / 512x1024 | 2:3 / 1:2 |
| 场景背景 | 1024x512 / 1344x768 | 16:9 |
| 物品图标 | 128x128 / 256x256 | 1:1 |

## 8. 高级用法

### 8.1 风格混合

将两种风格混合，创造独特的混合风格：

```
做法:
1. 准备两张不同风格的参考图
2. 在 reference_images 中同时传入两张
3. prompt 中描述混合比例：
   "结合{风格A}的色彩基调和{风格B}的笔触质感"

示例:
"结合浮世绘的鲜艳平面色块和赛博朋克的霓虹光效，
创造东方科幻融合风格"
```

### 8.2 风格渐变

在一组素材中实现风格渐变效果（如关卡 1→关卡 5 风格逐步变化）：

```
做法:
1. 定义起始风格和终止风格
2. 计算中间风格的六维度插值
3. 为每个阶段生成不同强度的风格描述

示例（5 关卡从明亮到黑暗）:
关卡 1: "明亮温暖色调，柔光，欢快氛围"
关卡 2: "温暖略沉色调，自然光，平静氛围"
关卡 3: "中性色调，侧光，紧张氛围"
关卡 4: "冷色调低饱和，暗光，压抑氛围"
关卡 5: "深暗冷色调，极暗光，恐怖氛围"
```

## 10. 注意事项

### 10.1 不要做的事

1. **不要跳过风格确认**：始终先让用户确认风格方向
2. **不要对 UI 元素过度风格化**：可读性优先于美观
3. **不要忽视透明通道**：图标和 UI 元素需要保持透明背景
4. **不要硬编码像素尺寸**：根据素材用途选择合适的 target_size
5. **不要修改 urhox-libs/ 目录**：这是只读的引擎库

### 10.2 必须做的事

1. **定义风格六维度**：每次迁移前完成风格分析
2. **建立风格锚定**：确保批量迁移的一致性
3. **展示对比**：原图 vs 风格化结果
4. **检查一致性**：完成后校验所有素材风格统一
5. **存储到 assets/**：所有生成的素材放在 assets/ 目录
6. **2 的幂次尺寸**：纹理贴图推荐 256/512/1024
