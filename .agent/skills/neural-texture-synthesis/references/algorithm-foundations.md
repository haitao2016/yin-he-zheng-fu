# Neural Style Transfer 算法基础

> 本文档提供 NST 算法的数学推导和理论基础，帮助开发者理解
> `neural-texture-synthesis` skill 各模块背后的原理。

---

## 1. 核心论文

**Gatys et al. (2015)** — *"A Neural Algorithm of Artistic Style"*

核心思想：用预训练卷积神经网络（VGG19）的中间层特征来分别捕获图像的
**内容信息**和**风格信息**，再通过优化生成图像使其同时匹配目标内容和风格。

### 1.1 VGG19 网络结构（简化）

```
输入图像 (224×224×3)
  ↓
Block 1: conv1_1 → relu1_1 → conv1_2 → relu1_2 → pool1     (64 filters)
Block 2: conv2_1 → relu2_1 → conv2_2 → relu2_2 → pool2     (128 filters)
Block 3: conv3_1 → relu3_1 → conv3_2 → relu3_2 → ... → pool3 (256 filters)
Block 4: conv4_1 → relu4_1 → conv4_2 → relu4_2 → ... → pool4 (512 filters)
Block 5: conv5_1 → relu5_1 → conv5_2 → relu5_2 → ... → pool5 (512 filters)
```

**关键层的作用**：

| 层 | 捕获的信息 | NST 中的用途 |
|----|-----------|-------------|
| relu1_1, relu2_1 | 低级特征（边缘、颜色） | 风格的精细纹理 |
| relu3_1 | 中级特征（纹理模式） | 风格的重复模式 |
| relu4_1 | 高级特征（物体部件） | 风格的抽象结构 |
| relu4_2 | 高级语义信息 | **内容表示** |
| relu5_1 | 最高级抽象 | 风格的全局组织 |

### 1.2 为什么选择 VGG19

- 结构简单（全部为 3×3 卷积 + 最大池化）
- 各层特征表示从具体到抽象，层次分明
- 预训练权重在 ImageNet 上学到了丰富的视觉先验

## 2. 特征提取与表示

### 2.1 特征图（Feature Map）

给定输入图像 x，VGG 第 l 层产生特征图 F^l：

```
F^l ∈ R^{N_l × M_l}

其中：
  N_l = 该层滤波器数量（通道数）
  M_l = 特征图的空间维度（高 × 宽）
  F^l_ij = 第 i 个滤波器在位置 j 的激活值
```

### 2.2 内容表示

内容信息直接用某一层（通常 relu4_2）的特征图表示：

```
Content(x) = F^l(x)    其中 l = relu4_2
```

选择较深层是因为：
- 保留了图像的语义结构（物体位置、形状）
- 丢弃了精细的像素级细节
- 对内容的表示更加抽象和鲁棒

### 2.3 Gram 矩阵（风格表示）

**这是 NST 最核心的数学工具。**

Gram 矩阵 G^l 定义为特征图的内积矩阵：

```
G^l_ij = Σ_k F^l_ik × F^l_jk

其中：
  G^l ∈ R^{N_l × N_l}
  i, j = 滤波器索引（1 到 N_l）
  k = 空间位置索引（1 到 M_l）
```

**直观理解**：
- G^l_ij 衡量滤波器 i 和滤波器 j 的**共激活程度**
- 如果两个滤波器经常在相同位置同时激活 → G_ij 值大
- 这捕获了特征之间的**统计相关性**，即"纹理模式"

**为什么 Gram 矩阵能表示风格**：
- 它丢弃了空间位置信息（通过对 k 求和）
- 保留了特征间的相关结构
- 相同风格的不同图像有相似的 Gram 矩阵
- 与图像的具体内容（物体位置）解耦

### 2.4 Gram 矩阵的推导

更严格地说，Gram 矩阵是特征图与其转置的乘积：

```
G^l = F^l × (F^l)^T / M_l

其中除以 M_l 是归一化因子，确保不同尺寸的层可以比较
```

展开写：

```
         M_l
G^l_ij = Σ   F^l(i,k) × F^l(j,k) / M_l
         k=1
```

这本质上是两个滤波器响应向量的**内积**（或相关系数）。

## 3. 损失函数

### 3.1 内容损失（Content Loss）

衡量生成图像 g 与内容图像 c 在特征空间的差异：

```
L_content(c, g) = (1/2) × Σ_ij (F^l_ij(g) - F^l_ij(c))²

其中 l 通常选择 relu4_2
```

**梯度**：

```
∂L_content/∂F^l_ij(g) = F^l_ij(g) - F^l_ij(c)    当 F^l_ij(g) > 0
                       = 0                          otherwise
```

### 3.2 单层风格损失

对于第 l 层，风格损失衡量 Gram 矩阵的差异：

```
E_l(s, g) = 1/(4 × N_l² × M_l²) × Σ_ij (G^l_ij(g) - G^l_ij(s))²

其中：
  s = 风格图像
  g = 生成图像
  N_l = 滤波器数量
  M_l = 空间维度
```

### 3.3 总风格损失（多层加权）

```
L_style(s, g) = Σ_l  w_l × E_l(s, g)

其中 w_l 是每层的权重，典型设置：
  w_relu1_1 = 1/5 = 0.2
  w_relu2_1 = 1/5 = 0.2
  w_relu3_1 = 1/5 = 0.2
  w_relu4_1 = 1/5 = 0.2
  w_relu5_1 = 1/5 = 0.2
```

### 3.4 总变差损失（Total Variation Loss）

用于生成图像的平滑性，减少噪声：

```
L_tv(g) = Σ_ij [(g(i+1,j) - g(i,j))² + (g(i,j+1) - g(i,j))²]
```

这惩罚相邻像素之间的剧烈变化。

### 3.5 总损失

```
L_total = α × L_content + β × L_style + γ × L_tv

典型权重：
  α = 1        (内容权重)
  β = 1000     (风格权重，通常远大于内容权重)
  γ = 0.001    (平滑权重)

α/β 比值控制内容-风格平衡：
  α/β = 1/10    → 强风格，弱内容
  α/β = 1/1000  → 非常强的风格化
  α/β = 1/100   → 平衡（推荐起点）
```

## 4. 优化过程

### 4.1 迭代优化

NST 使用基于梯度的优化（L-BFGS 或 Adam）来最小化总损失：

```
初始化：g ← 白噪声图像 或 内容图像的副本

重复 N 次迭代：
  1. 前向传播 g 通过 VGG，获得各层特征
  2. 计算内容损失、风格损失、总变差损失
  3. 反向传播计算梯度 ∂L_total/∂g
  4. 更新图像 g ← g - lr × ∂L_total/∂g
```

### 4.2 收敛特性

- 通常 300-1000 次迭代可以得到较好结果
- L-BFGS 收敛更快但内存占用大
- Adam 更稳定但需要更多迭代
- 内容损失通常先快速下降再缓慢收敛
- 风格损失下降更慢但更平稳

### 4.3 初始化策略

| 初始化方式 | 优点 | 缺点 |
|-----------|------|------|
| 白噪声 | 风格表现更自由 | 收敛慢，结果变化大 |
| 内容图像副本 | 收敛快，内容保留好 | 风格化可能不够彻底 |
| 风格图像副本 | 风格保留好 | 内容可能丢失 |

## 5. NST 到 Lua 的概念映射

由于 UrhoX Lua 无法运行 VGG19 神经网络，我们将 NST 的核心**数学思想**
映射为可在游戏中实时运行的 Lua 算法：

### 5.1 映射关系

| NST 概念 | 数学本质 | Lua 模块 | 实现方式 |
|----------|---------|---------|---------|
| 内容/风格分离 | 特征空间解耦 | ColorStatistics | RGB↔Lab 色彩空间转换 |
| Gram 矩阵 | 纹理统计相关 | ProceduralTexture | 噪声函数的频率/振幅参数 |
| 多层特征 | 多尺度分析 | PainterlyRenderer | 粗→细多层渲染 |
| 损失函数 | 差异度量 | ConsistencyScorer | CIE76 ΔE 色差 + 统计一致性 |
| 梯度下降 | 迭代优化 | ParameterOptimizer | 无梯度搜索 + 模拟退火 |

### 5.2 设计哲学

```
VGG19 特征提取  →  不可用（无 GPU 推理）
                →  替代：色彩统计 + 噪声函数 + 绘制规则

Gram 矩阵      →  不可用（无法计算）
                →  替代：Perlin/Worley 噪声参数化纹理模式

反向传播        →  不可用（无自动微分）
                →  替代：随机搜索 + 模拟退火

像素级优化      →  不可用（性能不足）
                →  替代：参数级优化（少量关键参数）
```

## 6. 色彩理论补充

### 6.1 CIE Lab 色彩空间

Lab 是感知均匀的色彩空间，使得欧氏距离近似人眼感知差异：

```
RGB → XYZ 转换：
  先将 sRGB 线性化：
    r_lin = (r > 0.04045) ? ((r+0.055)/1.055)^2.4 : r/12.92

  再应用矩阵（D65 光源）：
    X = 0.4124564×R + 0.3575761×G + 0.1804375×B
    Y = 0.2126729×R + 0.7151522×G + 0.0721750×B
    Z = 0.0193339×R + 0.1191920×G + 0.9503041×B

XYZ → Lab 转换：
  f(t) = t^(1/3)           当 t > (6/29)^3
       = t/(3×(6/29)^2) + 4/29  否则

  L = 116 × f(Y/Yn) - 16
  a = 500 × (f(X/Xn) - f(Y/Yn))
  b = 200 × (f(Y/Yn) - f(Z/Zn))

  其中 Xn, Yn, Zn 是 D65 白点：
    Xn = 0.95047, Yn = 1.00000, Zn = 1.08883
```

### 6.2 Reinhard 颜色迁移

在 Lab 空间中进行统计匹配：

```
对每个通道 c ∈ {L, a, b}：
  1. 计算源图像统计：μ_src, σ_src
  2. 计算目标图像统计：μ_tgt, σ_tgt
  3. 转换：pixel_out = (pixel_in - μ_src) × (σ_tgt / σ_src) + μ_tgt
```

这保持了像素间的相对关系，同时匹配了全局色彩分布。

### 6.3 CIE76 色差公式

```
ΔE_76 = √((L1-L2)² + (a1-a2)² + (b1-b2)²)

感知阈值：
  ΔE < 1    : 人眼不可分辨
  ΔE 1-2    : 近距离细看可辨
  ΔE 2-3.5  : 中等差异
  ΔE 3.5-5  : 明显差异
  ΔE > 5    : 显著差异
```

## 7. 噪声函数理论

### 7.1 Perlin 噪声

Ken Perlin (1985) 提出的梯度噪声：

```
核心思想：
  1. 在整数网格点放置随机梯度向量
  2. 对于任意坐标，找到周围4个网格点（2D）
  3. 计算到每个网格点的距离向量与梯度的点积
  4. 用缓和曲线插值这4个值

缓和函数（改进版）：
  fade(t) = 6t^5 - 15t^4 + 10t^3

性质：
  - 值域约 [-1, 1]（2D 约 [-0.707, 0.707]）
  - C² 连续（改进版）
  - 无明显方向性偏好（使用随机排列表）
```

### 7.2 分形布朗运动（fBm）

叠加多个不同频率的噪声层：

```
fBm(x) = Σ_{i=0}^{octaves-1} amplitude_i × noise(frequency_i × x)

其中：
  frequency_i = lacunarity^i × base_frequency
  amplitude_i = gain^i

典型参数：
  octaves = 6        (叠加层数)
  lacunarity = 2.0   (频率倍增因子)
  gain = 0.5         (振幅衰减因子)
```

### 7.3 Worley 噪声（细胞噪声）

Steven Worley (1996) 提出的基于距离场的噪声：

```
算法：
  1. 在空间中随机散布特征点（种子点）
  2. 对于任意位置，计算到最近 N 个特征点的距离
  3. F1 = 最近距离，F2 = 第二近距离
  4. 组合方式：F1, F2-F1, F2×F1 等产生不同效果

效果：
  F1        → 细胞形态（石墙、鳞片）
  F2 - F1   → 细胞边缘（裂缝、脉络）
  F1 × F2   → 混合效果
```
