# REVIEW 阶段 — 三维度审查清单

> 本文档是 Team Dev Skill REVIEW 阶段的详细审查参考。
> 每个维度包含具体检查项、常见问题和判断标准。

---

## 维度 1: 正确性审查

### 1.1 功能完整性

- [ ] 所有验收条件（`acceptanceCriteria`）逐条验证通过
- [ ] 边界场景测试：空输入、极大值、极小值
- [ ] 用户交互路径完整：正常流程 + 异常流程

### 1.2 边界与安全

- [ ] nil 值保护：所有外部输入在使用前检查 nil
- [ ] 数组边界：索引从 1 开始（规则 #4）
- [ ] 除零保护：涉及除法的地方检查分母
- [ ] 字符串安全：`tostring()` 转换后再拼接

### 1.3 内存安全

**反模式检测**（以下代码不应出现在每帧更新中）：

```
❌ HandleUpdate 中调用:
   - CreateChild()
   - CreateComponent()
   - Material:new()
   - SubscribeToEvent()（应在 Start 中订阅一次）
   - nvgCreateFont()（应在初始化时调用一次）
```

- [ ] 无每帧对象创建
- [ ] 无重复事件订阅
- [ ] 节点/组件创建在初始化阶段完成

### 1.4 逻辑正确

- [ ] 条件判断：`and`/`or` 优先级正确
- [ ] 循环边界：`for i = 1, n` 而非 `for i = 0, n-1`
- [ ] 状态转换：无死锁、无不可达状态
- [ ] 计时器/冷却：使用 `dt` 累加而非帧计数

## 维度 2: 引擎合规审查

### 2.1 核心规则快速检查

| 规则 | 检查方法 | 常见错误 |
|------|---------|---------|
| #0 长度单位(米) | 检查 position/scale 数值 | 角色高度 100（应为 1.5-2.0） |
| #0.5 坐标系 | Y=上, Z=前, X=右 | 混淆 Y/Z 轴 |
| #1 代码存放 | 文件在 `scripts/` 目录 | 放在根目录或其他位置 |
| #3 eventData | `GetInt()`/`GetFloat()` | 直接 `eventData.Key` |
| #4 数组索引 | 从 1 开始 | `array[0]` |
| #6 NanoVG 事件 | `NanoVGRender` | 在 Update 中绘制 |
| #7 NanoVG 字体 | `nvgCreateFont` 一次 | 每帧创建字体 |
| #9.1 模型尺寸 | `boundingBox` 或查文档 | 硬编码猜测尺寸 |
| #9.6 材质路径 | `PBRNoTexture` 系列 | 猜测 Technique 路径 |
| #10 UI 系统 | `urhox-libs/UI` | 使用原生 UIElement |
| #12 枚举值 | `MOUSEB_LEFT` 等 | 使用数字 0, 1, 2 |

### 2.2 自动检测模式

审查时可用以下关键词搜索代码中的违规：

```
数字常量替代枚举:  button == 0, button == 1, button ~= 0
数组从0开始:       [0], i = 0,
原生UI:           UIElement, CreateWindow, CreateButton
错误eventData:     eventData.Key, eventData["Key"]（缺少 GetXxx()）
每帧创建:         在HandleUpdate中搜索 CreateChild, CreateComponent, new()
```

## 维度 3: 游戏品质审查

### 3.1 可玩性

- [ ] 操作响应及时（无明显输入延迟）
- [ ] 物理参数合理（移动速度 3-10 m/s，跳跃初速 5-8 m/s）
- [ ] 碰撞检测准确（无穿模、无误判）

### 3.2 视觉效果

- [ ] 无 Z-fighting（重叠面闪烁，偏移至少 0.01）
- [ ] 无模型穿模
- [ ] 相机无穿墙
- [ ] UI 元素不遮挡关键游戏区域

### 3.3 性能

- [ ] 无每帧字符串拼接大量日志
- [ ] 无冗余的 `cache:GetResource()` 调用（应缓存引用）
- [ ] 粒子/特效有合理数量上限
- [ ] 大量实体使用对象池复用

### 3.4 代码质量

- [ ] 变量命名语义化（非 a, b, c, temp1, temp2）
- [ ] 函数职责单一，长度合理（< 100 行）
- [ ] 模块间依赖清晰，无循环依赖
- [ ] 类型标注完整（规则 #11）：nil 变量有 `---@type` 注解
