# 游戏设计模式与引导问题库

## 概述

本文档为交互式游戏设计教练提供：
1. 按游戏类型分类的设计模式速查
2. 引导提问的问题库
3. ASCII 可视化模板
4. UrhoX 脚手架映射规则

---

## 一、游戏类型识别与设计模式

### 1. 2D 休闲 / 街机

**识别关键词**: 休闲、街机、Flappy Bird、贪吃蛇、打砖块、弹球

| 设计维度 | 典型选择 |
|---------|---------|
| 视角 | 侧视 / 俯视 |
| 操作 | 单键 / 双键 / 滑动 |
| 核心循环 | 闪避、收集、匹配 |
| 胜负 | 无尽模式 + 计分 / 限时 |
| 进度 | 最高分排行榜 |

**推荐脚手架**: `templates/scaffold-2d.lua`
**关键技术**: NanoVG 渲染、`NanoVGRender` 事件

**默认推测**:
- 竖屏（手机优先）
- 触屏点击/滑动操作
- 无尽模式 + 分数排名

### 2. 2D 平台跳跃

**识别关键词**: 马里奥、平台跳跃、横版过关、Celeste、Hollow Knight

| 设计维度 | 典型选择 |
|---------|---------|
| 视角 | 横版侧视 |
| 操作 | 移动 + 跳跃 + 攻击 |
| 核心循环 | 移动、跳跃、战斗、收集 |
| 胜负 | 通关制 / 到达终点 |
| 进度 | 关卡解锁 |

**推荐脚手架**: `templates/scaffold-2d-physics.lua`
**关键技术**: Box2D 物理、碰撞检测、`center` 偏移

**默认推测**:
- 横屏
- 键鼠：WASD/方向键移动 + 空格跳跃
- 触屏：虚拟摇杆 + 跳跃按钮

### 3. 射击游戏（2D）

**识别关键词**: 射击、太空射击、弹幕、雷电、太空侵略者

| 设计维度 | 典型选择 |
|---------|---------|
| 视角 | 纵版（竖屏）/ 横版 / 俯视 |
| 操作 | 移动 + 射击（自动/手动） |
| 核心循环 | 移动闪避、射击消灭、收集强化 |
| 胜负 | 无尽 + 最高分 / 固定关卡 |
| 进度 | 武器升级、道具收集 |

**推荐脚手架**: `templates/scaffold-2d.lua`（无物理碰撞时）或 `templates/scaffold-2d-physics.lua`（需要物理时）

### 4. 益智 / 消除

**识别关键词**: 消消乐、俄罗斯方块、2048、拼图、三消

| 设计维度 | 典型选择 |
|---------|---------|
| 视角 | 正面 / 俯视（棋盘式） |
| 操作 | 点击 / 拖拽 / 滑动 |
| 核心循环 | 观察、决策、匹配、消除 |
| 胜负 | 得分目标 / 步数限制 / 时间限制 |
| 进度 | 关卡递进难度 |

**推荐脚手架**: `templates/scaffold-2d.lua`
**关键技术**: 网格数据结构、动画过渡

### 5. 3D 动作 / 冒险

**识别关键词**: 第三人称、动作游戏、冒险、砍杀、跑酷

| 设计维度 | 典型选择 |
|---------|---------|
| 视角 | 第三人称（TPS） |
| 操作 | WASD移动 + 鼠标视角 + 攻击 |
| 核心循环 | 探索、战斗、收集、成长 |
| 胜负 | 关卡通关 / 击败Boss |
| 进度 | 等级、装备、技能树 |

**推荐脚手架**: `templates/scaffold-3d-character.lua`
**关键技术**: 动画状态机（FSM）、ThirdPersonCamera、PBR 材质
**注意**: 鼠标模式设为 `MM_RELATIVE`

### 6. 3D 场景 / 模拟

**识别关键词**: 场景展示、建造、模拟、沙盒、Minecraft

| 设计维度 | 典型选择 |
|---------|---------|
| 视角 | 自由相机 / 第一人称 |
| 操作 | 自由移动 + 放置/建造 |
| 核心循环 | 探索、收集、建造 |
| 胜负 | 开放式 / 自定义目标 |
| 进度 | 资源积累、区域解锁 |

**推荐脚手架**: `templates/scaffold-3d-scene.lua`

## 二、引导提问库

### 核心问题（优先级排序）

按优先级从高到低，选择第一个尚未回答的问题：

| # | 问题 | 目的 | 可推测条件 |
|---|------|------|-----------|
| 1 | 玩家每秒钟在做什么？ | 确定核心循环 | 已知游戏类型时可推测 |
| 2 | 怎样算赢/输/游戏结束？ | 确定胜负条件 | 无尽类可推测为"死亡结束" |
| 3 | 玩家控制什么？ | 确定玩家实体 | 从类型+参考游戏推测 |
| 4 | 怎么操作？ | 确定操控方式 | 从平台+类型推测 |
| 5 | 场景里还有什么？ | 补充游戏对象 | 从类型推测常见对象 |
| 6 | 看起来什么样？ | 确定视觉风格 | 可用默认风格 |

### 深入问题（当核心问题已满足，用户想继续完善时）

| 维度 | 问题示例 |
|------|---------|
| 难度曲线 | 难度怎么递增？线性增加/阶梯式/随机波动？ |
| 道具系统 | 有没有可收集的道具/增强？ |
| 音效反馈 | 打击感怎么表现？屏幕震动/粒子效果/音效？ |
| 社交 | 需要排行榜吗？单机还是在线排名？ |
| 多人 | 是单人还是多人？合作还是对抗？ |
| 商业化 | 需要考虑广告位/内购吗？ |

## 三、ASCII 可视化模板库

### 竖屏模板（9:16）

```
┌──────────────┐
│  UI区域       │
│              │
│  游戏区域     │
│              │
│              │
│              │
│  控制区域     │
└──────────────┘
```

### 横屏模板（16:9）

```
┌────────────────────────────────────┐
│  UI                                │
│                                    │
│            游戏区域                 │
│                                    │
│  控制提示                           │
└────────────────────────────────────┘
```

### 常用 ASCII 符号

| 符号 | 含义 |
|------|------|
| `[P]` | 玩家 |
| `[E]` | 敌人 |
| `[B]` | Boss |
| `★` / `☆` | 道具/星星 |
| `♥` | 生命 |
| `●` / `○` | 子弹 |
| `█` | 墙壁/障碍 |
| `═` | 地面/平台 |
| `~` | 水面 |
| `☄` | 陨石/飞行物 |
| `⚡` | 闪电/能量 |
| `🚀` | 火箭/飞船 |
| `→` `←` `↑` `↓` | 方向 |

## 四、UrhoX 脚手架决策树

```
用户的游戏类型
  │
  ├─ 2D + 无物理需求 ──→ scaffold-2d.lua
  │    (休闲、街机、卡牌、消除)
  │
  ├─ 2D + 需要物理 ───→ scaffold-2d-physics.lua
  │    (平台跳跃、弹球、愤怒的小鸟)
  │
  ├─ 3D + 无角色 ────→ scaffold-3d-scene.lua
  │    (场景展示、建造模拟、策略)
  │
  └─ 3D + 有角色 ────→ scaffold-3d-character.lua
       (动作、冒险、射击、RPG)
```

## 五、从设计到开发的衔接清单

GDD 完成后，进入编码阶段前确认：

- [ ] 脚手架已选定
- [ ] 代码将放在 `scripts/` 目录
- [ ] 明确是单机还是多人（检查 `.project/settings.json` 的 `multiplayer.enabled`）
- [ ] 横屏/竖屏已确定
- [ ] 如需排行榜/云存档，确认使用 `clientCloud` API
- [ ] 如需 FPS/TPS 视角控制，确认设置 `input.mouseMode = MM_RELATIVE`
- [ ] 如需第三人称相机，使用 `ThirdPersonCamera` 库
- [ ] 构建工具准备就绪（每次修改后必须调用）
