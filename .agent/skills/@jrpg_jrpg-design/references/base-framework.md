# JRPG 通用设计框架

> 所有 JRPG 子类型共享的骨架。各子类型在此基础上调整权重和侧重。

---

## 设计总纲生成规则

每个 JRPG 项目必须有 **5 条设计总纲**。总纲是全局约束，后续所有系统围绕总纲展开。

生成总纲时：
1. 从子类型参考文件获取 3 条预设总纲（子类型核心）
2. 补充 2 条通用总纲（从下方库中选取或自拟）
3. 每条总纲必须在 ≥2 个系统中被兑现

### 通用总纲库（可选）

| ID | 总纲 | 适用场景 |
|----|------|---------|
| G1 | **丰而不强制** | 70-80% 内容可跳过；强制的只有主线最小路径 |
| G2 | **奖励知识，不奖励时间** | 乘算倍率让"懂得多"远比"磨得久"高效 |
| G3 | **无孤立系统** | 每个系统至少与 2 个其他系统有连接点 |
| G4 | **同一内容多层体验** | 每个元素至少服务两个目的 |
| G5 | **经济有入口也有出口** | 来源多样，去向也多样 |
| G6 | **失败是学习，不是惩罚** | 败北零损失 + 败北有收获 |
| G7 | **节奏即叙事** | 系统节奏本身讲述故事 |
| G8 | **每一步都有反馈** | 玩家的任何行动都能感知到结果 |

## 十大系统模块

以下是 JRPG 的十大系统模块。每个子类型对模块的**权重**不同（见子类型参考文件），但所有模块都应存在于最终文档中。

### 模块 0: 项目参数总表

先锁定数字，再展开系统。必填参数：

| 参数 | 说明 | 子类型影响 |
|------|------|-----------|
| 表游戏时长 | 通关前时长 | 故事型 20-60h / 箱庭 15-40h / 刷刷 50-300h |
| 里游戏时长 | 通关后时长 | 故事型 ≤表游戏 / 箱庭 ≈表游戏 / 刷刷 ≥表游戏 |
| 角色池规模 | 可用角色数 | 故事型 8-15 / 箱庭 1-6 / 刷刷 30-100 |
| 战斗队人数 | 同时上场数 | 通常 3-4 |
| 属性种类数 | 元素/属性体系 | 故事型 4-6 / 箱庭 0-4 / 刷刷 6-10 |
| 地图区域数 | 可探索区域 | 故事型 10-20 / 箱庭 5-12（深度大） / 刷刷 15-30 |
| 主线 Boss 数 | 关键战斗数 | 故事型 8-15 / 箱庭 6-10 / 刷刷 10-20 |
| 货币种类 | 经济复杂度 | 故事型 1-2 / 箱庭 1-3 / 刷刷 3-5 |

### 模块 1: 内容分层

定义必须层 / 推荐层 / 可选层的占比和内容条目。

**输出格式**：
```
| 层级 | 目标占比 | 内容条目 |
|------|---------|---------|
| 必须层 | 「待填:占比%」 | 「待填:具体内容，如主线任务、关键Boss」 |
| 推荐层 | 「待填:占比%」 | 「待填:具体内容，如支线、城镇设施」 |
| 可选层 | 「待填:占比%」 | 「待填:具体内容，如隐藏区域、收集」 |
```

加上表里分界（通关触发条件、通关后解锁内容）。

### 模块 2: 经济系统

核心约束：每种资源必须有 ≥2 入口 + ≥2 出口。

子模块：
- 2.1 资源定义表（ID / 名称 / 入口 / 出口）
- 2.2 平衡杠杆（零产出内容、银行隔离、被动收入等）
- 2.3 倍率乘算公式（如适用）
- 2.4 素材阶梯（如适用）
- 2.5 放置型设施（如适用）
- 2.6 城镇投资（如适用）

### 模块 3: 战斗系统

子模块：
- 3.1 战斗类型（回合制 / 即时 / 混合）与核心循环
- 3.2 属性体系（相克关系 + 安全阀）
- 3.3 Boss 设计模板（行动周期 / 弱点窗口 / 规则修改器）
- 3.4 难度曲线

### 模块 4: 角色与成长

子模块：
- 4.1 角色池结构（获取方式 / 独占能力 / 类型配比）
- 4.2 角色分配位（战斗队 / 预备队 / 生产设施等）
- 4.3 成长机制（等级 / 转生 / 技能树 / 觉醒 等）
- 4.4 相性系统（如适用）
- 4.5 效率质变点

### 模块 5: 装备与合成

子模块：
- 5.1 装备槽位定义
- 5.2 合成体系（统一造价 / 素材链）
- 5.3 装备决策点（异常免疫拼图 / 属性取舍）
- 5.4 入手阶段变化

### 模块 6: 探索与地图

子模块：
- 6.1 区域定义表（解锁条件 / 内容描述）
- 6.2 时间系统（昼夜 / 季节 / 天气，如适用）
- 6.3 隐藏要素规则
- 6.4 特色探索机制（零战斗迷宫 / 密语收集 / 环境解谜等）
- 6.5 传送体系
- 6.6 Boss 链（如适用）

### 模块 7: 任务与系统联动

核心约束：任务系统是枢纽，连接所有系统。

子模块：
- 7.1 跨系统前置（解锁内容需满足多系统条件）
- 7.2 序列门控
- 7.3 行为累积成就
- 7.4 多条件交叉任务
- 7.5 贯穿元素（≥1 元素贯穿 ≥3 系统）

### 模块 8: 玩家心理保障

核心公式：失败代价→0 + 挑战奖励恒正 → 玩家永远倾向"去试试"。

子模块：
- 8.1 可逆/不可逆清单（不可逆 ≤0.1%）
- 8.2 多层体验事件（负面→反转→精通）
- 8.3 难度调节（诈称 / 自适应 / 手动选择）
- 8.4 安全网设计

### 模块 9: 通关后与长线留存

子模块：
- 9.1 终盘挑战内容（制约迷宫 / 竞技场 / 隐藏 Boss 等）
- 9.2 日课/循环内容
- 9.3 挑战即报酬机制
- 9.4 二周目 / NG+ 设计（如适用）

### 模块 10: 系统耦合矩阵

8×8 矩阵验证所有系统互联。每行每列 ≥2 连接，否则该系统趋于孤立。

## 系统接口规范

每个模块末尾必须附带系统接口表：

```
| 输出 | → 目标系统 | 说明 |
|------|-----------|------|
| XXX  | 模块 N    | 描述 |
```

确保信息流向清晰，无孤立系统。

## 附录模板

### A: 设计检验总清单

按模块生成 checklist，每条对应一个可验证的设计标准。格式：
```
### 模块名
- [ ] 检验条目（具体、可判 Yes/No）
```

### B: 文档维护

```
| 项 | 值 |
|----|----|
| 创建日期 | 「待填」 |
| 最后更新 | 「待填」 |
| 子类型 | 「待填」 |
| 迭代规则 | 完成一个系统→填写全部占位符→更新耦合矩阵→跑总清单 |
```

---

## 占位符约定

| 占位符 | 含义 | 示例 |
|--------|------|------|
| `「待填」` | 需用户根据世界观/游戏内容填写 | `主角：「待填」` |
| `「待填:提示」` | 带引导的待填项 | `「待填:第一幕核心冲突」` |
| `「可调:默认值」` | 可调数值，已有推荐值 | `通关时长：「可调:50h」` |

禁止使用裸 `___` 或无类型标记的占位符，确保语义明确。

## Layer 2 导出约定

设计文档中的以下数据应可直接提取为 Lua 配置表（GameDesign table）：

| 模块 | 可提取数据 | 对应 Lua 路径 |
|------|-----------|--------------|
| 模块 0 | 项目参数总表所有数值 | `GameDesign.params.*` |
| 模块 2 | 资源定义表、倍率公式、素材阶梯 | `GameDesign.economy.*` |
| 模块 3 | 伤害公式、属性相克矩阵、评价倍率 | `GameDesign.battle.*` |
| 模块 4 | 经验公式、转生参数、角色池结构 | `GameDesign.growth.*` |
| 模块 5 | 装备槽位定义、合成阶梯 | `GameDesign.equipment.*` |
| 模块 6 | 昼夜参数、区域定义表 | `GameDesign.exploration.*` |

设计文档中使用 `「可调:X」` 标记的数值在导出时以 X 作为默认值。
