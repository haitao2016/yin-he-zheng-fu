# 故事驱动 JRPG 设计参考

> 代表作：Final Fantasy VI/VII/X、Persona 4/5、Xenoblade Chronicles、Chrono Trigger、Dragon Quest V
>
> 核心体验：**玩家为了"接下来会发生什么"而推进游戏**。系统服务于叙事节奏，战斗服务于戏剧张力。

---

## 子类型识别关键词

用户需求中出现以下词汇时，选择此子类型：
- 故事驱动、剧情向、叙事、角色弧、多结局、分支剧情
- Persona 风格、FF 风格、感人、催泪、命运
- 对话系统、好感度、日常系统、社交链

---

## 预设总纲（取 3 条）

| # | 总纲 | 一句话 |
|---|------|--------|
| S1 | **叙事即奖励** | 最好的奖励不是数值，是"接下来的故事"；剧情推进本身就是最强激励 |
| S2 | **角色即系统** | 每个可操作角色携带独占叙事线 + 独占机制；失去角色 = 失去玩法维度 |
| S3 | **节奏即叙事** | 战斗密度、探索节奏、系统解锁时机全部服从叙事弧线；高潮前收紧，喘息时展开 |

搭配通用总纲建议：G4（同一内容多层体验）、G6（失败是学习不是惩罚）

## 模块权重矩阵

| 模块 | 权重 | 设计侧重 |
|------|------|---------|
| 0 项目参数 | ★★★ | 时长以叙事章节衡量，非磨时间 |
| 1 内容分层 | ★★★★★ | **核心**。必须层≈30-40%（主线叙事不能太薄），可选层提供叙事深度而非重复内容 |
| 2 经济系统 | ★★ | 轻量。金钱不构成主要游戏循环，避免磨钱节奏破坏叙事 |
| 3 战斗系统 | ★★★ | 中等。战斗服务于叙事高潮，Boss 战是剧情节点不是数值检查 |
| 4 角色与成长 | ★★★★★ | **核心**。角色弧线 = 成长弧线；机制解锁绑定剧情进展 |
| 5 装备与合成 | ★★ | 轻量。简化合成，重装备叙事意义（传说之剑 > 通用强化路线） |
| 6 探索与地图 | ★★★ | 中等。地图服务于叙事场景转换，环境叙事比隐藏收集品更重要 |
| 7 任务与联动 | ★★★★ | 高。支线 = 角色支线故事，不是跑腿任务 |
| 8 玩家心理 | ★★★★ | 高。情感投入是核心资产，叙事选择的"重量感"需精心设计 |
| 9 通关后留存 | ★★ | 轻量。故事讲完即完整；NG+/多结局比无限刷重要 |
| 10 耦合矩阵 | ★★★ | 叙事线是最大的耦合枢纽 |

## 模块特化指南

### 模块 1: 内容分层（故事型特化）

**结构：章节制**

```
| 章节 | 时长 | 叙事功能 | 系统解锁 |
|------|------|---------|---------|
| 序章 | 1-2h | 世界观铺垫、核心角色登场 | 基础移动+战斗 |
| 第一幕 | 5-8h | 冲突建立、队伍扩张 | 探索系统、支线开放 |
| 第二幕 | 8-15h | 冲突升级、角色弧高潮 | 合成、特殊系统 |
| 转折 | 2-3h | 剧情反转、失去→重获 | 可能临时失去系统 |
| 第三幕 | 5-10h | 最终决战准备 | 全系统开放 |
| 终章 | 2-4h | 高潮+结局 | 无新系统 |
```

**关键规则**：
- 主线叙事场景**不可跳过**（与"丰而不强制"的平衡：支线可跳过，主线不行）
- 每章至少 1 个"选择时刻"——不一定影响结局，但让玩家感到参与
- 失去与重获：中后期必须有一次"暂时失去能力/同伴"的体验

**表里分界**：
- 通关 = 主线故事结束（有明确终点）
- 里游戏 = 角色个人结局补完 + 隐藏剧情 + NG+（非无限循环）

### 模块 2: 经济系统（故事型特化）

**简化原则**：经济不应成为叙事的阻力。

- 资源种类：1-2 种主货币即可
- 战斗掉落金钱足够支撑主线消费，无需刷钱
- **叙事经济**：关键道具通过剧情获得而非购买
- 倍率乘算：不适用或极简化
- 城镇投资：替换为"城镇重建"叙事（投入→可视变化→NPC 反馈）

### 模块 3: 战斗系统（故事型特化）

**战斗即叙事场景**：

Boss 设计模板增加叙事字段：

```
| 字段 | 规格 |
|------|------|
| 叙事定位 | 这场战斗在故事中的意义（复仇/保护/觉醒/告别） |
| 情感基调 | 悲壮/热血/绝望→希望/平静 |
| 台词触发 | 战中特定条件触发角色对话/回忆 |
| 败北叙事 | 首次败北是否有专属剧情（推荐有） |
| 演出时机 | 战斗中插入的过场动画节点 |
```

**难度哲学**：
- 主线 Boss 允许 2-3 次尝试通过（不应成为叙事堵塞点）
- 可选 Boss 难度不封顶
- 提供"故事模式"降低战斗难度

### 模块 4: 角色与成长（故事型特化）

**角色 = 独立叙事线**：

每个可操作角色必须具备：

| 要素 | 说明 |
|------|------|
| 个人动机 | 为什么在队伍里（不是"英雄救世"的通用理由） |
| 角色弧 | 从 A 状态变为 B 状态的内在转变 |
| 独占任务链 | 2-5 个专属支线任务 |
| 独占机制 | 只有该角色能做的事（不仅是战斗，可以是探索/交互/解谜） |
| 关键选择 | ≥1 个影响该角色结局的选择 |

**成长绑定叙事**：
- 能力解锁 = 剧情里程碑（不是纯等级阈值）
- 例：角色觉醒新能力 ← 面对个人创伤的剧情场景
- 转生机制替换为"觉醒阶段"：每次觉醒对应一次角色弧转折

**队伍规模**：8-15 个深度角色 > 50 个浅层角色

### 模块 5: 装备与合成（故事型特化）

**叙事装备 > 数值装备**：

- 关键装备通过剧情获得（传说之剑、母亲的遗物、敌人的武器）
- 装备带有 flavor text 讲述背景故事
- 合成系统简化为 1-2 步线性流程
- **角色专属装备**绑定角色支线完成度

### 模块 6: 探索与地图（故事型特化）

**环境叙事优先**：

| 探索要素 | 故事型实现 |
|---------|-----------|
| 隐藏区域 | 蕴含世界观补充（日记/壁画/废墟），不是空洞的宝箱 |
| NPC 对话 | 随主线章节变化，反映世界状态 |
| 回访价值 | 旧区域在新章节有新内容（NPC 台词变化/新事件） |
| 环境变化 | 剧情推进引发地图物理变化（城镇被毁/季节变化） |

### 模块 7: 任务与联动（故事型特化）

**支线 = 角色故事**：

支线任务分类：
| 类型 | 占比 | 特征 |
|------|------|------|
| 角色支线 | 60% | 绑定特定角色，推进角色弧 |
| 世界观支线 | 25% | 补充世界背景，无特定角色绑定 |
| 挑战支线 | 15% | 纯玩法挑战（竞技场/限时任务） |

**禁止**：无叙事意义的跑腿/收集任务（"帮我收集 10 个狼皮"）

### 模块 8: 玩家心理（故事型特化）

**情感设计**：

| 设计维度 | 方法 |
|---------|------|
| 选择的重量感 | 选择前给足信息 + 选择后有不可逆叙事后果（但不影响通关） |
| 告别设计 | 至少 1 次强制分离（角色暂时/永久离队），赋予情感冲击 |
| 沉默时刻 | 高潮后安排安静段落（无战斗/无任务/纯探索+对话） |
| 玩家代入 | 主角有足够性格但留有代入空间 |

**安全网调整**：
- 故事模式（可选）：战斗伤害减半，不影响叙事体验
- 多存档位：鼓励回看选择分支
- 剧情回放：已看过的关键剧情可重看

### 模块 9: 通关后（故事型特化）

**故事讲完 = 游戏完整**，通关后是**补完**而非**循环**：

| 通关后内容 | 类型 |
|-----------|------|
| 角色个人结局 | 补完未完成的角色支线 |
| 隐藏结局条件 | 满足特定条件解锁的不同结局 |
| NG+ | 保留等级/装备，可做不同选择 |
| 开发者房间 | 彩蛋/致谢/花絮 |

**不适合**：无限制约迷宫、日课系统、击破数强化

## 特有系统：对话与分支

故事驱动 JRPG 需要额外的对话系统设计：

### 对话系统

| 参数 | 推荐值 |
|------|--------|
| 选择频率 | 每 30-60 分钟 1 次有意义选择 |
| 选项数 | 2-4 个（避免选择疲劳） |
| 影响层级 | 即时反馈（NPC 反应）→ 短期（任务走向）→ 长期（结局） |
| 假选择占比 | ≤20%（结果相同但对话不同的选择，用于代入感） |

### 好感度/社交系统（可选）

| 参数 | 推荐值 |
|------|--------|
| 好感度等级 | 5-10 级 |
| 升级方式 | 对话选择 + 支线完成 + 战斗配合（非礼物轰炸） |
| 解锁内容 | 专属对话 → 专属技能 → 联携技 → 个人结局 |

## 推荐视觉风格

故事驱动型以情感传达为核心，视觉服务于叙事氛围。

| 维度 | 推荐方向 | 说明 |
|------|---------|------|
| 画风 | 手绘/赛璐璐/水彩 | 强调角色表情和情感表达，避免过度写实 |
| 视角 | 俯视45°（探索）+ 电影式特写（剧情） | 剧情场景切换电影镜头语言 |
| 色彩策略 | 按章节情感变化调色板 | 序章暖色→转折冷色→终章回暖 |
| 角色设计 | 高辨识度剪影 + 情感化配色 | 每个角色有标志色，配色反映性格 |
| UI 风格 | 书页/羊皮纸/日记本 | 融入叙事世界观，避免科技感 UI |

**色板参考**：

| 章节阶段 | 主色调 | 情绪 |
|---------|--------|------|
| 序章/日常 | 暖黄、天蓝 | 安宁、温馨 |
| 冲突升级 | 深红、灰紫 | 紧张、不安 |
| 转折/低谷 | 冷蓝、灰黑 | 悲伤、绝望 |
| 高潮/希望 | 金色、白色 | 热血、希望 |
| 结局/余韵 | 柔橙、淡紫 | 释然、怀念 |

## 推荐音频风格

故事驱动型音乐以情感渲染为核心，配合叙事节奏起伏。

| 场景 | 风格关键词 | AI 作曲提示词参考 |
|------|-----------|-----------------|
| 主菜单 | orchestral, majestic, nostalgic | `orchestral JRPG main theme, majestic yet nostalgic, strings and piano` |
| 城镇/日常 | acoustic, warm, folk | `warm acoustic town theme, folk guitar, peaceful, gentle flute` |
| 探索/野外 | orchestral, adventure, curious | `adventure exploration, light orchestral, curious woodwinds` |
| 普通战斗 | rock + orchestral, energetic | `energetic battle theme, rock guitar with orchestral strings, fast tempo` |
| Boss 战 | epic, choir, dramatic | `epic boss battle, dramatic orchestra, choir, intense percussion` |
| 剧情高潮 | piano, strings, emotional | `emotional cinematic piano and strings, bittersweet, building to climax` |
| 喘息/日常交流 | piano solo, ambient, gentle | `gentle piano solo, ambient, reflective, quiet moment` |
| 告别/悲伤 | solo instrument, minimal | `sorrowful solo violin, minimal accompaniment, slow, heartbreaking` |

**音乐设计原则**：
- 角色主题曲：每个主要角色有专属旋律动机（leitmotif），在相关剧情中变奏出现
- 场景调性匹配：高潮段使用大调/属七增强张力，低谷段使用小调/减七
- 沉默即演出：关键转折点可使用无BGM（纯环境音），强化冲击力
