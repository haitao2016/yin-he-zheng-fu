# 箱庭 JRPG 设计参考

> 代表作：塞尔达传说系列、Metroid（带 RPG 成长的变体）、CrossCode、天穗之咲稻姬、圣剑传说、伊苏系列
>
> 核心体验：**玩家为了"那边有什么"而推进游戏**。探索驱动一切——新能力打开新区域，新区域产出新能力，形成正向发现循环。

---

## 子类型识别关键词

用户需求中出现以下词汇时，选择此子类型：
- 箱庭、银河城/Metroidvania、互联地图、能力门控
- 塞尔达风格、探索驱动、开放世界（小规模）
- 解谜、环境交互、道具驱动探索
- 非线性、自由路线、序列破坏

## 预设总纲（取 3 条）

| # | 总纲 | 一句话 |
|---|------|--------|
| B1 | **地图即谜题** | 整个世界是一个嵌套的大型谜题；每个区域既是独立谜题又是大谜题的一部分 |
| B2 | **能力即钥匙** | 每个新能力至少打开 2 个之前"看得见够不着"的通路；能力获取 = 世界展开 |
| B3 | **密度胜于面积** | 每 10 步必有可交互要素；空旷 = 设计失败 |

搭配通用总纲建议：G3（无孤立系统）、G8（每一步都有反馈）

## 模块特化指南

### 模块 1: 内容分层（箱庭特化）

**结构：区域制（非章节制）**

```
| 区域 | 可达条件 | 核心能力需求 | 隐藏深度 |
|------|---------|-------------|---------|
| 起始区域 | 初始 | 无 | 3 层（表面/地下/高处） |
| 区域 A | 能力 1 | 二段跳 | 2 层 |
| 区域 B | 能力 2 | 水下呼吸 | 4 层 |
| 枢纽区域 | 能力 1+2 | 组合运用 | 5 层（最深） |
```

**关键规则**：
- 每个区域有"表面通过"和"完全探索"两种深度
- 主路径只需表面通过（~30% 内容）
- 100% 探索需所有能力 + 回访

**可选内容的密度而非数量**：
- 故事型的可选 = 支线剧情（内容量）
- 箱庭型的可选 = 同一区域的隐藏层级（密度层）

### 模块 2: 经济系统（箱庭特化）

**探索即经济**：

| 资源 | 入口 | 出口 |
|------|------|------|
| 通用货币 | 探索发现、战斗掉落 | 能力升级、装备购买 |
| 区域特产 | 特定区域独有 | 跨区域合成配方 |
| 能力碎片 | 隐藏位置收集 | 能力强化（量变→质变） |

**特有机制**：
- **快捷道（Shortcut）经济**：发现捷径 = 时间节省 = 隐性收益
- **地图补全奖励**：区域地图完成度达到阈值给予实质奖励

### 模块 3: 战斗系统（箱庭特化）

**即时战斗 + 环境交互**：

| 设计要点 | 说明 |
|---------|------|
| 战斗发生在探索中 | 无独立战斗场景，敌人在地图上 |
| 环境利用 | 可推落/点燃/冻结/利用地形 |
| 能力双用 | 探索能力在战斗中也有用（二段跳闪避/水下能力压制火敌） |
| Boss = 区域考试 | 检验该区域教授的能力组合 |

**Boss 设计模板增加字段**：

```
| 字段 | 规格 |
|------|------|
| 能力检验 | 该 Boss 检验哪些能力的掌握 |
| 竞技场设计 | Boss 房间的地形布局（可交互元素） |
| 解谜阶段 | Boss 战中的非战斗阶段（机关/环境互动） |
| 二次利用 | Boss 竞技场在战后是否变为新探索区域 |
```

### 模块 4: 角色与成长（箱庭特化）

**成长 = 能力扩展，不是数值膨胀**：

| 成长类型 | 说明 | 示例 |
|---------|------|------|
| 新能力（水平扩展） | 可以做之前不能做的事 | 二段跳、潜水、磁力 |
| 能力强化（垂直深化） | 已有能力更强/更远/更持久 | 跳跃高度+50%、潜水时间翻倍 |
| 数值成长 | 血量/攻击力增加 | 被动增长，占比≤30% |

**角色池规模**：1-6 个可操作角色（少而深）
- 单主角 + 能力扩展（塞尔达模式）
- 2-4 主角各自拥有独占能力（可切换）
- 角色获取 = 能力获取 = 世界展开

### 模块 5: 装备与合成（箱庭特化）

**装备改变玩法而非数值**：

| 装备类型 | 效果 | 示例 |
|---------|------|------|
| 移动装备 | 改变移动方式 | 滑翔伞、攀爬手套、潜水服 |
| 战斗装备 | 改变战斗策略 | 远程弓、近战剑、魔法杖 |
| 探索装备 | 改变环境交互 | 透视镜、磁力手套、缩小帽 |

**合成 = 区域特产组合**：鼓励跨区域探索

### 模块 6: 探索与地图（箱庭特化）★核心★

**地图互联设计法**：

#### 6.1 区域互联矩阵

```
| 从╲到 | 区域A | 区域B | 区域C | 枢纽 |
|-------|-------|-------|-------|------|
| 区域A | -     | 能力2 | ×     | 初始 |
| 区域B | 能力1 | -     | 能力3 | 能力2 |
| 区域C | ×     | 能力3 | -     | 能力1+2 |
| 枢纽  | 初始  | 能力2 | 能力1+2 | - |
```

每个单元格 = 从 A 到 B 所需的能力。`×` = 不直连。

#### 6.2 可见不可达设计

每个区域至少设置 3 个"看得见但当前到不了"的点：

```
| 位置 | 可见时机 | 可达条件 | 回访奖励 |
|------|---------|---------|---------|
| 高台宝箱 | 初次进入 | 二段跳（区域B后） | 能力碎片×3 |
| 水下洞穴 | 初次进入 | 潜水（区域C后） | 隐藏 Boss |
| 暗门 | 获得透视后 | 透视+炸弹（后期） | 最终装备材料 |
```

#### 6.3 快捷道系统

```
| 快捷道类型 | 开启方式 | 效果 |
|-----------|---------|------|
| 单向捷径 | 从终点侧打开 | 回访时跳过前半段 |
| 双向传送 | 双端同时激活 | 跨区域快速移动 |
| 隐藏通道 | 特定能力揭示 | 跳过整个区域 |
```

#### 6.4 垂直性

箱庭地图必须有垂直维度：
- 每个区域至少 2 个垂直层级（地面/高处 或 地面/地下）
- 垂直移动能力是重要的成长里程碑

### 模块 7: 任务与联动（箱庭特化）

**环境谜题 = 隐式任务**：

| 任务类型 | 占比 | 特征 |
|---------|------|------|
| 环境谜题 | 50% | 无 NPC 指引，纯观察+推理+能力运用 |
| NPC 委托 | 25% | 指向特定区域的探索目标 |
| 收集达成 | 15% | 区域内碎片收集→能力升级 |
| Boss 挑战 | 10% | 区域 Boss + 隐藏 Boss |

**序列破坏（Sequence Breaking）**：
- 是否允许：建议**有限允许**
- 预设 2-3 条非预期路线，给予彩蛋奖励
- 不允许完全破坏核心体验的序列破坏

### 模块 8: 玩家心理（箱庭特化）

**"啊哈时刻"设计**：

| 类型 | 设计方法 |
|------|---------|
| 捷径发现 | 绕了一大圈后发现回到起点旁边 |
| 能力回访 | 新能力让旧区域展现全新面貌 |
| 连锁解锁 | 一把钥匙打开三扇门 |
| 视角转换 | 从高处俯瞰走过的区域，理解全局结构 |

**地图焦虑缓解**：
- 渐进式地图揭示（走过的路自动标记）
- 区域完成度百分比显示
- 无限时限制（除非是可选挑战）

### 模块 9: 通关后（箱庭特化）

| 通关后内容 | 说明 |
|-----------|------|
| 100% 完成度 | 补全所有隐藏区域/收集品 |
| 挑战模式 | 限时通过/无伤通过/最少步数 |
| Boss Rush | 连续 Boss 战 |
| 隐藏最终区域 | 100% 后解锁的终极区域 |

## 特有系统：能力门控设计

### 能力获取时间线

```
| 能力 | 获取区域 | 探索用途 | 战斗用途 | 解锁区域 |
|------|---------|---------|---------|---------|
| 能力1 | 起始 | 破坏脆弱墙壁 | 近战攻击 | 区域A的隐藏部分 |
| 能力2 | 区域A | 水上移动 | 水属性攻击 | 区域B |
| 能力3 | 区域B | 高处到达 | 空中攻击 | 区域C + 区域A回访 |
```

### 能力组合矩阵

| 能力A╲能力B | 跳跃 | 潜水 | 磁力 |
|-------------|------|------|------|
| 跳跃 | - | 水面跳→水下 | 磁力跳远 |
| 潜水 | 水下跳出 | - | 水下磁吸 |
| 磁力 | 同上 | 同上 | - |

每种 2 能力组合至少对应 1 种环境谜题类型。

## 地图设计检验

- [ ] 每个区域至少 3 个"可见不可达"点？
- [ ] 每个新能力至少解锁 2 个旧区域的新通路？
- [ ] 主路径不超过总内容 30%？
- [ ] 每 10 步（游戏内距离单位）有可交互要素？
- [ ] 存在至少 2 条非线性通过路线？
- [ ] 区域互联矩阵无死胡同区域？

## 推荐视觉风格

箱庭型以空间感和探索引导为核心，视觉服务于地图可读性。

| 维度 | 推荐方向 | 说明 |
|------|---------|------|
| 画风 | 像素/Low-Poly/手绘卡通 | 清晰的色块边界便于区分可交互要素 |
| 视角 | 俯视45°/横版 | 地图互联结构的可读性优先 |
| 色彩策略 | 按区域分配主色调 | 每个区域独立色调，玩家一眼识别所在位置 |
| 环境设计 | 高密度可交互标识 | 可交互物体有统一视觉提示（微光/颜色差异） |
| UI 风格 | 简洁/HUD 最小化 | 不遮挡探索视野，地图信息集成到环境中 |

**色板参考**：

| 区域类型 | 主色调 | 环境氛围 |
|---------|--------|---------|
| 起始区/安全区 | 翠绿、天蓝 | 安全、引导 |
| 森林/自然 | 深绿、棕色 | 探索、神秘 |
| 水域/冰原 | 蓝白、青色 | 清澈、危险 |
| 火山/危险区 | 橙红、黑色 | 威胁、挑战 |
| 遗迹/神殿 | 金色、紫色 | 古老、庄严 |
| 隐藏区域 | 与周围区域形成对比 | 惊喜、发现 |

## 推荐音频风格

箱庭型音乐以区域辨识和氛围营造为核心，强调空间感。

| 场景 | 风格关键词 | AI 作曲提示词参考 |
|------|-----------|-----------------|
| 主菜单 | adventure, mysterious, inviting | `adventure JRPG menu theme, mysterious and inviting, orchestral` |
| 起始区域 | folk, warm, simple | `simple folk melody, warm and welcoming, acoustic instruments` |
| 森林区域 | ambient, nature, woodwinds | `forest exploration, ambient nature sounds, soft woodwinds, mysterious` |
| 水域区域 | ethereal, flowing, harp | `underwater/water area theme, ethereal, flowing harp, ambient pads` |
| 危险区域 | tense, percussion, low brass | `tense danger zone, low brass, tribal percussion, ominous` |
| 遗迹/神殿 | reverent, echo, choir | `ancient ruins theme, reverent choir, echoing, grand and mysterious` |
| 普通战斗 | action, fast, rhythmic | `action battle theme, fast-paced, rhythmic percussion, energetic` |
| Boss 战 | intense, phase-shifting | `intense boss battle, shifts between phases, building intensity` |
| 发现/解谜 | playful, music box, curious | `puzzle discovery jingle, playful music box, curious, rewarding` |
| 能力获得 | triumphant, fanfare, short | `ability unlock fanfare, triumphant, short celebratory brass` |

**音乐设计原则**：
- 区域主题曲：每个区域有独立旋律，跨区域时平滑过渡（crossfade）
- 垂直层叠：同一区域的地面/地下/高处使用同主题不同编曲（加减乐器层）
- 发现音效：每种"啊哈时刻"有专属短旋律（捷径发现/隐藏房间/新能力）
