## 1. 项目概述

**银河征服** V1.6 在 V1.5 全功能版本基础上，聚焦三大方向：
- **科技树深化**：解锁新舰种、建筑特化分支，让科研决策更有战略重量
- **星球事件扩展**：随机事件增加链式结果、负面事件和特殊地点，提升探索趣味
- **多人对战体验**：战斗后排行榜、跨玩家成就对比，强化社交竞争感

**技术栈**：UrhoX 引擎 + raw NanoVG 渲染 + 多人网络（Client/Server 架构）

## 2. 现状分析（V1.5 基线）

### 代码规模

| 文件 | 行数 | 说明 |
|------|------|------|
| `GalaxyScene.lua` | 3099 | 银河地图主场景 |
| `BattleScene.lua` | 2683 | 战斗场景（含阵型/MVP/单舰面板）|
| `Client.lua` | 2587 | 客户端主逻辑（含存档/事件/UI） |
| `GameUI.lua` | 2450 | UI主模块（含成就墙/日志/市场） |
| `Systems.lua` | 1319 | 数据定义（含科技树/舰种/市场） |
| `GalaxyEvents.lua` | 419 | 随机事件（8种，3原有+5新增）|
| `BattleSkills.lua` | 432 | 战斗主动技能（6种）|
| `AchievementSystem.lua` | 248 | 成就系统（18个成就）|

### 识别的可改进点

1. **科技树只有 4 Tier 13 项**，无分支选择，后期缺乏策略差异化
2. **随机事件 8 种**，缺少链式事件（事件引发新事件）和负面事件（危机处理）
3. **成就只有 18 个**，缺少战斗专项成就（无伤/连击/MVP）和隐藏成就
4. **战斗无环境效果**：所有波次地形一致，缺乏nebula（增伤）、asteroid（掩护）等环境变量
5. **无跨会话战绩统计**：缺少生涯胜负/最高波次/累计击杀等历史数据面板
6. **星球面板建筑槽可视化不足**：当前只显示已建建筑列表，缺少"可建但未建"槽位预览
7. **舰队生产无优先级队列UI**：只能看到倒计时，无法拖拽调整建造顺序

## 5. 功能详细说明

### P1-1 科技树分支选择

**背景**：当前 Tier3 有 3 项科技（WARP_DRIVE、ADVANCED_WEAPONS、DEFENSE_MATRIX），研究完可全解锁，无策略选择压力。

**方案**：
- Tier3 新增 2 条**互斥分支**：选择「进攻路线」则解锁 `NOVA_CANNON`（全体AOE+40%），但永久锁死 `DEFENSE_MATRIX`
- 选择「防御路线」解锁 `FORTRESS_PROTOCOL`（基地护盾翻倍+回血），但锁死 `ADVANCED_WEAPONS`
- TechPanel 显示分支提示，已选分支变灰并标注"已锁定"
- 存档中记录已选分支，保证跨局唯一性

### P1-2 战斗环境效果

**背景**：所有波次战场完全相同，缺乏变化感。

**3种环境类型**（每场战斗随机1种）：
| 环境 | 视觉效果 | 游戏效果 |
|------|---------|---------|
| 星云（Nebula） | 紫色粒子背景 | 全体攻击力+20%，护盾恢复速度+30% |
| 小行星带（Asteroid Belt）| 漂浮岩石障碍 | 己方可躲避岩石阻挡敌方视线 |
| 磁暴区（Magnetic Storm） | 蓝白闪电特效 | 技能冷却速度+50%，但受伤概率+10% |

- `BattleScene.Init()` 随机选择环境类型，存入 `battleEnv_`
- 渲染层在背景绘制后叠加环境粒子/图形
- 战斗开始弹出"当前战场：★ 星云区域"提示横幅

### P1-3 链式随机事件

**背景**：当前所有事件为独立单次事件，选择后即结束。

**链式机制**：
- 为 3 种现有事件（MERCHANT、RIFT、MINE）添加 `chain` 字段
- 选择特定选项后，60秒内在附近生成后续子事件（奖励升级或触发危机）
- 例：选择"进入时空裂缝采集" → 60秒后在附近生成"裂缝扩散"危机事件（不处理则扣除基地护盾）
- 链式图标：原事件图标右下角显示小锁链符号"⛓"

### P2-1 生涯战绩面板

**数据字段**（新增到存档）：
```lua
careerStats = {
    totalGames    = 0,  -- 总游戏局数
    totalWins     = 0,  -- 胜利次数
    bestWave      = 0,  -- 历史最高战斗波次
    totalKills    = 0,  -- 生涯总击杀
    totalColonies = 0,  -- 生涯总殖民数
    bestMvpShip   = "", -- 历史最强MVP舰种
    playtime      = 0,  -- 累计游戏时长（秒）
}
```
- 新增"战绩"Tab按钮（在成就旁）
- 面板6格数据卡片布局（3×2），每格图标+数字+标签

### P2-2 建筑槽可视化

**背景**：当前 PlanetPanel 和 BasePanel 只显示已建建筑，空槽不可见。

**方案**：
- 按 `BaseModuleSlots(coreLevel)` 计算最大槽位数，将已建+未建全部渲染
- 空槽显示灰色虚线框 + "+" 号 + 悬停显示可建建筑列表
- 点击空槽直接打开对应建造确认弹窗，减少操作层级

### P2-3 舰队建造队列UI

**背景**：当前 FleetPanel 只有一个倒计时进度条，无法管理多个排队项目。

**方案**：
- 显示最多3个排队中的舰船（图标+名称+进度条）
- 每个排队项右侧有 "×" 取消按钮和 "↑" 上移优先级按钮
- 动画：当前建造中的舰船图标闪烁蓝光

  P1-2  战斗环境效果    BattleScene 视觉+逻辑，压轴

  P2-2  建筑槽可视化    提升日常操作流畅度

## 8. 风险与注意事项

| 功能 | 风险 | 说明 |
|------|------|------|
| P1-1 科技分支 | 中 | 互斥逻辑需注意存档兼容性，老存档需初始化分支为 nil（无选择状态）|
| P1-2 战场环境 | 中 | 需测试环境效果不破坏现有 Boss 波次平衡（磁暴+技能CD加速不能过强）|
| P1-3 链式事件 | 低 | GalaxyEvents 模块独立，只需新增 chain 字段，不影响现有事件流程 |
| P2-1 战绩面板 | 低 | 纯新增存档字段+UI，老存档使用默认 0 值，无兼容问题 |
| P2-2 建筑槽 | 低-中 | PlanetPanel/BasePanel 高度需动态扩展，注意与 colonyAdvRowH 叠加 |
| P2-3 队列UI | 低 | FleetPanel 已有队列数据，主要是渲染层改造 |
| P3-x 润色 | 低 | 均为纯渲染层，无逻辑变更 |
