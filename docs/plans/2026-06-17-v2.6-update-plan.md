## 3. 关键功能详细设计

### P0-1 FleetPanel 新舰种解锁校验

**背景**：FleetPanel 的 `drawShipQueue()` 只检查 `SHIP_QUEUE_ORDER`，未校验新舰种的科技解锁条件。

**实现方案**：
- `GameConstants.lua` 补全 `SHIP_UNLOCK_REQUIREMENTS`：STEALTH→tech=PHASE_DRIVE，SUPPORT→tech=NANO_REPAIR，DREADNOUGHT→coreLevel=8
- FleetPanel 渲染舰船按钮时：若 `ship.stype` 在 SHIP_UNLOCK_REQUIREMENTS 中 → 检查 Client 是否满足条件；不满足则按钮灰色半透明，tooltip 显示解锁条件
- DREADNOUGHT 额外检查：`vars_.dreadBuiltThisWave = false`，波次开始时重置；建造后置 true，下一波次恢复

**新增状态**：
```lua
-- FleetPanel.lua / Client.lua
-- dreadBuiltThisWave_: bool, 每波次开始 Reset() 时置 false
```

### P0-3 多阶段 Boss 血条分段渲染

**背景**：`BOSS_PHASES` 表已定义各阶段 hpThreshold，但 BattleRender 血条为普通血条。

**实现方案**：
- BattleRender 检测 `ship.isBoss == true` 后，使用 `ship.bossPhases` 绘制分段血条（阶段数以竖线标记）
- 阶段转换时：bossPhaseChanged 标志 → 全屏横幅（"进入阶段 X：形态名"）+ 0.8s 无敌（ship.invulnTimer 已由 BattleAI 设置）
- 虚空 Boss 特殊：阶段 1 隐形状态仅显示"虚空 Boss · 隐形中"红色提示

**位置**：`BattleRender.drawBossHealthBar()` 或在 `drawUI()` 中的 Boss 血条渲染分支

### P1-1 战术协同（TACTICAL_SYNC）

**效果**：激活 8s 内，所有舰船攻击同一目标时伤害叠加 +8%/艘（上限 50%）；击杀目标后全队获得 2s 加速 ×1.5

**实现要点**：
- BattleSkills 新增 `TACTICAL_SYNC_CD = 60`、`TACTICAL_SYNC_DUR = 8`
- 新增 `tacticalSyncActive_ = 0` 状态
- BattleAI 的伤害计算中插入协同倍率：统计同目标友舰数 → `damageMult = 1 + min(0.5, allyCount * 0.08)`
- 目标死亡事件回调中触发全队加速 buff

### P1-2 引力陷阱（GRAVITY_TRAP）

**效果**：指定区域（半径 80）生成引力场，敌方舰船进入后移速 ×0.3 且无法离开；场内敌舰每秒受 10 伤害

**实现要点**：
- BattleScene 新增 `gravityTraps_ = {}` 列表（`{x,y,radius,life,maxLife}`）
- 玩家点击按钮后进入"选择区域"状态 → 点击战场空白处生成陷阱
- BattleRender 绘制圆形半透明紫色范围，带旋转粒子效果（NanoVG）
- BattleAI Update 中，每个 trap 对范围内敌舰应用减速 + 持续伤害

### P1-4 Boss 专属技能：要塞炮击 + 无人机群

**要塞炮击（战列Boss阶段2，10s CD）**：对血量最高友舰造成 200 伤害 + 击退 50px（屏幕震动效果）
- 实现：BattleAI UpdateEnemy 中检查 `ship.isBoss and ship.bossPhaseName == "狂怒形态"`，使用 `cannonTimer_` 倒计时

**无人机群（母舰Boss阶段1-2，8s CD）**：释放 6 架自爆无人机（HP20，接触友舰爆炸 40 伤害）
- 实现：`makeDroneShip()` 临时生成小型敌舰，自毁逻辑复用现有 drone 框架扩展

### P2-1 防御塔（DEFENSE_TURRET）

**效果**：该星球战斗中自动生成 2 座轨道炮（HP100，攻击 25，射程 300）；每升 1 级 +1 座

**实现要点**：
- `GameConstants.lua` 新增 BUILDINGS.DEFENSE_TURRET：cost={metal=400, esource=200}, buildTime=15, prod={}, specDefense={...}
- BuildingSystem 中该建筑效果在战斗开始时挂钩：BattleScene 的 `generateEnemyFleet()` 前调用 `injectDefenseTurrets(planet)` 生成友方防御塔
- PlanetPanel 新增建筑按钮（与 MINE/POWER_PLANT 等同行列表扩展）

### P2-4 星门节点（STARGATE_NODE）

**效果**：该星球可作为舰队瞬移目标点，60s 冷却

**实现要点**：
- `GameConstants.lua` BUILDINGS.STARGATE_NODE：cost={metal=800, esource=500, nuclear=200}, buildTime=25
- GalaxyScene 舰队移动逻辑中：若目标星球含 STARGATE_NODE 且我方基地核心已部署"主曲速门" → 瞬移（`obj.x = targetPlanet.x; obj.y = targetPlanet.y`）
- 星门冷却由 `GalaxyScene.stargateCooldowns_ = {[planetId] = timeLeft}` 管理

### P2-6 Tier5 科技

**实现要点**：
- `GameConstants.lua` 的 TECHS 中新增 6 项 Tier5 条目（恒星引擎/量子工厂/虚空舰队/要塞协议II/时空研究/文明飞跃），tier=5，各有独立效果
- TechPanel 中 tier==5 科技默认隐藏，仅当基地核心 Lv10 后解锁显示（金色边框 + 专属图标）
- ResearchSystem 中应用 Tier5 科技时写入全局效果（如 `globalResourceMult = 1.5`），由 ResourceManager 读取

## 7. 技术风险与缓解策略

| 风险 | 等级 | 说明 | 缓解措施 |
|------|------|------|---------|
| 新舰种解锁条件与现有科技树冲突 | 中 | SHIP_UNLOCK_REQUIREMENTS 中引用的 techId 可能不存在 | 在 GameConstants 顶部加 `CHECK_UNLOCK_REFS()` 启动期校验，未找到时日志报警 |
| 旧存档加载新字段时崩溃 | 高 | 老存档无 `rareResources` 字段 | `setmetatable({__index = function() return 0 end})` 或防御性 `rawget` fallback |
| Boss 多阶段状态机与现有 DDA 冲突 | 中 | 多阶段 Boss HP 下降速度受 DDA 影响，可能导致阶段转换过快/过慢 | 阶段转换触发后设置 `phaseChangeLockTimer = 3s`，锁定期内不允许再次转换 |
| 引力陷阱 NanoVG 渲染性能 | 中 | 半径 80 的范围渲染 + 旋转粒子，1080p 下无压力，但 4K 可能掉帧 | 使用 `nvgShapeAntiAlias(false)` 关闭抗锯齿；粒子数上限 12 |
| 星门节点与现有舰队移动逻辑冲突 | 低 | 舰队可能在移动过程中被瞬移，导致路径残留 | 瞬移前先设置 `obj.targetX = nil; obj.moveTarget = nil`，清空现有指令 |
| Tier5 科技效果与现有全局倍率叠加无上限 | 低 | 叠加导致数值溢出 | 所有全局倍率设置 `math.min(10.0, value)` 硬上限 |
