## 4. 详细方案说明

### P0-1: 战斗回放系统增强

**问题**: 当前战斗回放仅有基础播放功能，缺少时间轴控制和关键帧导航。

**方案**:
- **回放控制条**: 快进（1x/2x/4x）、慢放（0.5x）、暂停、跳转
- **关键帧标记**: 自动标记击杀、技能释放、阵型切换等关键事件
- **回放元数据**: 保存战斗摘要（持续时间、MVP、胜负、难度等）
- **回放列表**: 在战斗结算后保存最近 10 场战斗
- **导出功能**: 导出战斗数据为结构化 JSON

**核心文件**: `BattleReplayPlayer.lua` + 新文件 `BattleReplayControls.lua`
**规模**: ~200 行

---

### P0-2: 战斗损伤分析面板

**问题**: 玩家在战斗后缺少对战斗过程的详细分析，无法知道哪艘舰贡献最大。

**方案**:
- **伤害来源分布**: 按舰种/编队显示伤害输出百分比
- **受击类型统计**: 物理/能量/爆炸伤害分布
- **舰种贡献度**: 每艘舰船的具体贡献（伤害/承伤/治疗/辅助）
- **时间轴统计**: 按时间段（前 30 秒 / 中期 / 尾声）分别统计
- **雷达图展示**: 6 维度雷达图整合 MVP 数据

**核心文件**: `BattleStatsTracker.lua`（已有基础） + 新面板文件
**规模**: ~180 行

---

### P0-3: 指挥官技能树

**问题**: 当前指挥官系统仅有基础属性，缺少深度和成长感。

**方案**:
- **技能树结构**: 每个指挥官 3 分支（攻击/防御/辅助）+ 8 技能节点
- **技能节点类型**:
  - 被动技能：永久属性加成（+10% 舰队攻击等）
  - 主动技能：战斗中可激活（战术技能冷却管理复用 BattleCommandSystem）
  - 终极技能：解锁条件复杂，效果强力
- **技能点获取**: 指挥官等级提升获得技能点
- **技能重置**: 可花费资源重置技能树
- **技能树 UI**: 类科技树可视化，节点连接 + 解锁状态展示

**核心文件**: `CommanderSystem.lua` 扩展
**规模**: ~250 行

---

### P0-4: 动态市场扩展

**问题**: 当前 MarketSystem 仅有基础价格波动，缺少深度和玩家参与感。

**方案**:
- **稀有资源交易**: 蓝水晶/紫水晶等稀有资源开放交易
- **价格趋势图**: 显示最近 24 小时价格走势（折线图）
- **交易历史**: 玩家自己的买卖记录 + 收益统计
- **市场波动事件**: 随机事件影响特定资源价格（短缺/过剩/战争需求）
- **交易限额**: 每日交易量上限，防滥用

**核心文件**: `MarketSystem.lua`（已有基础） + 新 UI 面板
**规模**: ~150 行

---

### P0-5: 大文件拆分第二阶段

**问题**: 3 个超大文件严重影响维护效率：
- `GameUI.lua`: ~3112 行
- `Client.lua`: ~2733 行
- `BattleScene.lua`: ~2573 行

**方案**:
- **GameUI.lua** → 拆分为 `GalaxyUI.lua`（星图界面）+ `BattleUI.lua`（战斗界面）+ `CommonUI.lua`（通用面板）
- **Client.lua** → 拆分为 `ClientGameLoop.lua`（游戏主循环）+ `ClientDataManager.lua`（数据管理）
- **BattleScene.lua** → 依赖 `battle/` 子目录已有模块（BattleAI/BattleRender/BattleState），简化主入口为调度器
- **原则**: 单文件 ≤ 1500 行，单一职责原则
- **策略**: 逐文件拆分，每文件拆分后立即构建验证

**规模**: 结构重构（非新增代码），预计修改 500+ 行迁移代码

---

### P1-6: 类型注解专项

**问题**: Lua 作为动态类型语言，缺少类型注解导致开发效率低、错误难发现。

**方案**:
- **全量补充**: 所有 public 函数补充 `---@param` / `---@return` 注解
- **自定义类型**: 定义常用类型别名 `---@type FleetData` / `---@type PlanetData`
- **45+ 系统文件增量进行**:
  - 优先: `ResearchSystem.lua`、`FleetManager.lua`、`BaseBuildingSystem.lua`
  - 中期: `MarketSystem.lua`、`TradeSystem.lua`、`CommanderSystem.lua`
  - 后期: 其余 systems/ 文件

**规模**: 增量 ~500 行注解，分散 45+ 文件

---

### P1-7: 存档迁移工具

**问题**: 当前缺少从旧版本存档迁移到新版本的自动化工具，玩家升级后可能丢失进度。

**方案**:
- **SaveMigrationTool**:
  - 检测存档版本号（V1/V2）
  - V1 → V2 数据格式转换（键名补全）
  - V2 → V3 迁移（科技树重新映射）
  - **校验和重计算**: 迁移后重新生成校验和
  - **备份机制**: 迁移前自动备份原始存档
- **迁移 UI**: 在加载存档时自动检测并提示迁移
- **存档编辑器**: 开发者用的存档查看/修复工具

**核心文件**: 新文件 `SaveMigrationTool.lua` + `SaveSlotSystem.lua` 扩展
**规模**: ~120 行
