## 4. 详细方案说明

### M1-1: 大文件拆分 Phase 2

**目标文件**:
- `GameUI.lua` (~3112 行) → 拆分为 `GalaxyUI.lua` + `BattleUI.lua` + `CommonUI.lua`
- `Client.lua` (~2733 行) → 拆分为 `ClientGameLoop.lua` + `ClientDataManager.lua`
- `BattleScene.lua` (~2573 行) → 简化为调度器，battle/ 子目录已模块化

**拆分策略**:
1. 每个文件识别核心职责，建立新子文件
2. 原文件保留薄包装层（wrapper），所有原有 `require` 路径不变，确保向后兼容
3. 每拆完一个文件立即进行游戏内验证
4. 最终目标：单文件 ≤ 1500 行

**文件划分**:
```
GameUI.lua
  → GalaxyUI.lua      (星图 UI: 星球面板/舰队面板/资源面板)
  → BattleUI.lua      (战斗 UI: HUD/技能按钮/指令面板)
  → CommonUI.lua      (通用 UI: 设置/存档/通知/加载)

Client.lua
  → ClientGameLoop.lua   (主循环: Update/Render/Input)
  → ClientDataManager.lua (数据管理: 存档加载/同步/状态)

BattleScene.lua
  → 简化为 BattleOrchestrator.lua (主调度器)
  → 已有模块: BattleAI/ /BattleRender/ /BattleState/
```

**验收**: 所有原有 `require("game.GameUI")` 等路径不变，构建 0 错误，功能无回归

### M2-1: 战斗回放控制面板

**布局设计**:
```
┌─────────────────────────────────────────────────┐
│  回放: [玩家] vs [AI]  难度:困难  结果:胜利     │
│─────────────────────────────────────────────────│
│  [<<] [<] [▶/⏸] [>] [>>]  速度: [0.5x][1x][2x][4x] │
│  ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 1:23 │
│  ★ 关键帧: 击杀 #3 (0:15)  |  技能释放 (0:32)  |  击杀 #7 (0:55) │
│─────────────────────────────────────────────────│
│  [导出 JSON]  [删除回放]  [关闭]                │
└─────────────────────────────────────────────────┘
```

**核心功能**:
- 速度切换: 0.5x / 1x / 2x / 4x
- 逐帧控制: 上一帧 / 暂停 / 下一帧
- 关键帧跳转: 点击关键帧条目直接跳转
- 进度条拖动: 可拖动进度条跳转到任意时间点
- 导出: 生成结构化 JSON 供分析或分享
